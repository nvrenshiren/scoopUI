## 1. 项目概述

scoop-gui 是一个 Windows 桌面应用,为用户已经安装在本机的 Scoop(Windows 命令行包管理器)提供图形化操作界面,把命令行才能完成的软件包浏览、安装、卸载、更新、搜索、桶管理等操作转换为可视化、可点击的桌面端操作。

---

## 2. 业务目标

用户使用本产品要达成以下目的:

1. **降低使用门槛**:不必记忆 Scoop 命令及其参数,即可完成日常软件包管理操作。
2. **集中浏览**:在一个界面内统一查看本机已安装软件包、可安装软件包、有可用更新的软件包。
3. **快速检索**:通过搜索在已知桶中定位需要的软件包,而不必翻命令行搜索结果。
4. **可视化的桶管理**:用界面查看当前已添加的桶、添加新桶、移除桶,而非手动维护命令行配置。
5. **可观察的操作结果**:每次安装/卸载/更新/桶变更都有清晰的进度与结果反馈,失败时给出原因。
6. **中英双语言界面**:面向用户的所有文案支持中文与英文两套,首次启动可由用户选择并可在产品内随时切换。
7. **可配置的首次安装协助**:在协助用户安装 Scoop 时,把安装路径、全局目录、缓存目录、代理设置等关键参数以可填写的方式呈现给用户,而不是仅执行默认安装。

## 3. MVP 范围

### 3.1 本阶段(MVP 阶段一)做

- 浏览软件包:查看本机已装软件包列表,以及在已添加桶中、尚未安装到本机的可装软件包列表。
- 搜索软件包:在已添加桶范围内按关键字搜索软件包。
- 查看已装软件列表:展示已装软件包的版本信息,并对"存在更新版本"的软件包给出过期标记。
- 安装软件包:对单个软件包执行安装。
- 卸载软件包:对单个已装软件包执行卸载。
- 更新软件包:
  - 单包更新:对单个已装软件包升级到当前桶内的最新版本。
  - 批量更新所有:一次性更新本机所有存在过期标记的软件包。
- 桶管理:
  - 查看当前已添加的桶列表。
  - 查看 Scoop 已知的桶清单(用于添加时选择目标桶)。
  - 添加新桶到本机。
  - 从本机移除桶。
- 首次启动检测与安装协助:产品启动进入主流程前,检测本机 `scoop` 命令是否可用;未安装时,在产品内引导用户完成 Scoop 首次安装。具体包括:
  - 提供一键执行安装能力,安装过程中提供可观察的进度与结果反馈,失败时允许重试并以业务语言展示失败原因。
  - 以可填写的方式向用户呈现 Scoop 安装配置项(安装根目录 `ScoopDir`、全局软件包目录 `ScoopGlobalDir`、下载缓存目录 `ScoopCacheDir`、代理设置 `NoProxy` / `Proxy`、代理凭据 `ProxyCredential` / `ProxyUseDefaultCredentials`、是否以管理员权限运行 `RunAsAdmin`),默认值采用 Scoop 官方安装脚本 `install.ps1` 的默认值。
  - 每个配置项配以清晰的业务说明,让普通用户能理解其含义;当用户选择"全局软件包目录"等需要管理员权限的选项时,提示该选项需要管理员权限并协助其提权执行,但不强制要求管理员。
  - 所有配置项在执行安装前一次性确认,安装过程中不再允许改动;确认后的配置项在本机持久化,以便后续重新安装 Scoop 时复用。
- 中英双语言界面:产品面向用户的所有界面文案(菜单、按钮、提示信息、错误信息、进度文本等)必须同时具备中文与英文两套。具体包括:
  - 首次启动时允许用户选择界面语言(中文 / English 二选一),选择结果在本机持久化,后续启动按用户上次选择显示,不再询问。
  - 用户可在产品内随时切换界面语言,切换后立刻生效。
  - 不允许出现"半中半英"或"只有英文"的界面元素,所有面向用户的文案必须同时具备中英两套。
  - 仅支持中文与英文两种语言,不引入第三方语言包。

### 3.2 本阶段(MVP 阶段一)不做

以下能力**留待后续 spec 单独覆盖**,本阶段不予实现:

- 自动更新检查计划(定时/触发式的过期检查与提醒)
- 备份与还原(Scoop 备份/还原命令的图形化封装)
- 系统托盘与系统通知
- Scoop 配置项的编辑(配置查看/修改界面)
- shims 管理
- 国际化扩展:本产品只提供中文与英文两套界面文案,不引入第三方语言包、不实现运行时翻译;后续若需要更多语言,由独立 spec 单独覆盖

### 3.3 业务边界(项目级硬约束)

下列约束**不属于功能**,而是横贯全部业务规则的边界条件:

- **软件源唯一性**:本产品所展示与管理的所有软件包,均来源于用户本机已安装的 Scoop 及其已添加的桶;**不联网发现新的软件源**,不引入 Scoop 之外的第三方包索引。
- **单实例约束**:同一时刻,本机只允许运行一个 scoop-gui 进程;再次启动应触发对已运行实例的唤起/聚焦,而非并发拉起。
- **用户模型单一**:本产品只有一类用户,即"本机用户";不实现多用户体系、角色权限、登录鉴权。
- **界面语言范围**:本产品仅支持中文与英文两种界面语言,不引入第三方语言包,不实现运行时翻译;首启选择与后续切换均在中英两套文案之间进行。
- **安装可配置项边界**:产品的安装协助仅覆盖 Scoop 官方安装脚本 `install.ps1` 已声明的参数集(`ScoopDir` / `ScoopGlobalDir` / `ScoopCacheDir` / `NoProxy` / `Proxy` / `ProxyCredential` / `ProxyUseDefaultCredentials` / `RunAsAdmin`),不引入未在官方脚本中暴露的安装选项。
- **Scoop 自身不纳入管理范围**:本产品的**职责范围仅限"协助安装 Scoop 本身"这一步** —— 即首次启动时对本机 `scoop` 命令可用性的检测,以及在本产品内引导并执行 Scoop 首次安装的能力;此外,**不**对 Scoop 自身的卸载、升级、修复等维护操作负责。
  - 在"首次使用场景"下,本产品承担"检测 + 协助安装"的责任,确保用户在本产品内即可完成 Scoop 的首次部署。
  - 一旦 Scoop 被判定为"已可用",其它业务模块(浏览/搜索/安装/卸载/更新/桶管理等)即可正常使用,但仍以"Scoop 已可用"为前提继续运行 —— 本产品不会在运行过程中再次尝试介入 Scoop 自身的升级或修复。

## 6. 关键业务术语表(草稿)

> 本节仅列术语清单与一句话业务指向,详细定义见 `docs/prd/glossary.md`。

| 术语 | 业务指向 |
| --- | --- |
| 软件包 | Scoop 管理的一个具体应用或工具单位 |
| 桶 | 软件包所属的来源仓库,一个桶内含一组软件包及其版本元数据 |
| 已装软件包 | 已经通过 Scoop 安装到本机的软件包 |
| 可装软件包 | 在已添加桶中存在、但尚未安装到本机的软件包 |
| 过期 | 一个已装软件包的当前本机版本,落后于其所属桶中可用的最新版本 |
| 安装 | 将桶内的软件包部署到本机 |
| 卸载 | 将已装软件包从本机移除 |
| 更新 | 将已装软件包升级到其所属桶中的最新版本 |
| 更新所有 | 一次性更新本机所有处于过期状态的软件包 |
| 已知桶 | Scoop 自身维护的、用户可从中选择添加的桶清单 |
| 单实例 | 同一时刻本机只允许存在一个 scoop-gui 运行进程 |
| 本机用户 | 本产品的唯一使用角色,具备调用 `scoop` 命令的权限 |
| 首次启动检测 | scoop-gui 启动进入主流程前,对本机 `scoop` 命令是否可用的检查行为 |
| 协助安装 | 在 scoop-gui 产品内引导用户完成 Scoop 首次安装的能力,包括未安装时的入口引导、可配置项填写、过程进度反馈与失败重试 |
| 界面语言 | 本产品面向用户的所有界面文案所使用的语言(中文 / English 二选一) |
| 界面语言选择 | 用户在首次启动或后续使用中切换界面语言的行为,选择结果在本机持久化 |
| 安装配置项 | 安装 Scoop 时用户可填写的参数集合(路径、缓存、代理等),来源于官方 `install.ps1` 暴露的参数集 |
| 提权执行 | 以管理员权限执行安装动作,用于需要全局目录(`ScoopGlobalDir`)等场景 |
