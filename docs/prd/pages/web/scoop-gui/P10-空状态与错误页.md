# P10 · 空状态与错误页

## 1. 目的

为主界面及各业务页面在“暂无数据”“搜索无结果”“读取失败”“运行期间 Scoop 不可用”等情况下提供统一、友好的呈现。该页面作为子状态帮助用户理解当前问题,并给出明确下一步动作。

## 2. 功能清单

- 功能 1:F19 错误处理与友好提示 —— 对空状态、读取失败、命令级失败、网络失败、用户取消、运行期间 Scoop 整体不可用等情况给出业务语言提示。
- 功能 2:F17 安装任务进度反馈 —— 当错误来自长时任务失败时,承接 P08 的失败结果,提供重试或返回来源页的引导。

## 3. 页面流转

### 3.1 入口

本页面不是独立顶级入口,而是 P03 / P04 / P05 / P06 等页面的数据为空或读取失败时的子状态呈现。异常处理口径引用 `flows/scoop-gui.md` §1.9;运行期间 Scoop 整体不可用引用 `flows/scoop-gui.md` §1.2 异常分支。相关实体状态机只按来源页面引用 `flows/scoop-gui.md` §2 对应小节,本页不复述。

### 3.2 内部流转

识别当前子状态类型 → 展示对应业务提示 → 提供下一步动作,包括重试、清空搜索、返回主界面、前往桶管理或查看帮助 → 用户选择动作后返回来源页面或相关目标页面。

### 3.3 出口

- 用户选择重试时,返回来源页面重新读取或重新尝试当前动作,异常规则引用 `flows/scoop-gui.md` §1.9。
- 用户选择清空搜索时,返回 P05 的无查询浏览状态,对应 `flows/scoop-gui.md` §1.2 中搜索与浏览路径。
- 用户选择前往桶管理时,进入 P06,对应 `flows/scoop-gui.md` §1.7。
- 用户选择返回主界面时,回到 P03,对应 `flows/scoop-gui.md` §1.2。
- 若错误来自长时任务失败并选择重试,进入 P08,对应原动作所在的 §1.3 / §1.4 / §1.5 / §1.6 / §1.7 / §1.1。

## 4. 交互说明

- 场景 1:已装列表为空
  - 触发:P04 没有可展示的已装软件包。
  - 呈现:界面说明当前暂无已装软件包,并引导用户前往 P05 浏览可安装软件包。
  - 结果:用户知道下一步可以搜索或安装软件包。
- 场景 2:可装列表为空
  - 触发:P05 在无查询条件下没有可装软件包。
  - 呈现:界面说明当前可装范围为空,并引导用户前往 P06 添加桶或重试读取。
  - 结果:用户知道可通过桶管理扩展可装范围。
- 场景 3:搜索无结果
  - 触发:P05 输入查询后没有匹配项。
  - 呈现:界面说明没有匹配结果,并提供清空查询或修改查询的动作。
  - 结果:用户可以回到全部可装列表或继续调整搜索。
- 场景 4:桶列表为空
  - 触发:P06 已添加桶列表为空。
  - 呈现:界面说明当前尚未添加桶,并引导用户切换到已知桶清单添加。
  - 结果:用户可继续完成添加桶流程。
- 场景 5:读取失败
  - 触发:P03 / P04 / P05 / P06 任一区段数据无法读取。
  - 呈现:界面以业务语言说明读取失败,并提供重试或返回主界面的动作。
  - 结果:用户不会面对空白页面,可尝试恢复。
- 场景 6:运行期间 Scoop 不可用
  - 触发:进入主界面后,相关区段发现 Scoop 无法继续提供信息。
  - 呈现:界面说明当前无法读取 Scoop 数据,并提示这不是首次协助安装流程。
  - 结果:用户可重试读取或返回其它可用页面,产品不自动进入 P01。
- 场景 7:长时任务失败
  - 触发:P08 返回失败结果并需要在来源页展示后续处理。
  - 呈现:界面展示失败对象、失败原因与重试 / 放弃 / 返回的动作。
  - 结果:用户可继续处理失败项或回到来源页。

## 5. 验收要点

- ✅ P10 不应作为独立主导航页面出现,应作为 P03 / P04 / P05 / P06 等页面的子状态呈现。
- ✅ P04 已装列表为空时,应显示友好空状态,而不是空白区域。
- ✅ P05 无查询且可装范围为空时,应提示当前没有可装软件包,并引导用户前往桶管理或重试。
- ✅ P05 有查询但无匹配结果时,应提示没有搜索结果,并提供清空查询或修改查询的动作。
- ✅ P06 已添加桶为空时,应提示尚未添加桶,并引导用户查看已知桶清单。
- ✅ P06 已知桶清单读取失败时,应提示读取失败并提供重试。
- ✅ P03 主区读取失败时,应显示友好错误提示,不应导致整个产品不可操作。
- ✅ 运行期间 Scoop 整体不可用时,应提示无法读取相关信息,但不应自动进入 P01 协助安装。
- ✅ 每个空状态或错误状态都应提供至少一个明确下一步动作。
- ✅ 错误提示应使用业务语言说明用户能理解的影响,不应只展示难以理解的底层错误。
- ✅ 对可重试失败,用户应能再次尝试原动作或重新读取。
- ✅ 对用户取消导致的终止,提示不应把取消描述为失败。
- ✅ 长时任务失败返回来源页后,失败对象应可被识别,并提供重试或放弃路径。
- ✅ 空状态与错误状态下,语言应遵循当前界面语言选择,不得半中半英。
- ✅ P10 的引导动作不应引入本阶段未定义的功能,例如登录、配置编辑、备份还原或系统通知。
