# P02 · 语言选择页

## 1. 目的

在用户首次启动且本机尚未保存界面语言偏好时,让用户在中文与 English 之间做出明确选择。该选择用于决定后续所有面向用户文案的显示语言,并在本机持久化。

## 2. 功能清单

- 功能 1:F13 界面语言选择(首次启动) —— 首次启动时提供中文 / English 二选一,并在确认后立即生效与持久化。
- 功能 2:F19 错误处理与友好提示 —— 当语言选择保存失败时,以业务语言告知用户本次即时生效与下次启动可能回退的影响。

## 3. 页面流转

### 3.1 入口

本页面从 `flows/scoop-gui.md` §1.1 首次启动流程进入:启动序列进入“检测语言偏好”后,若 UI Language 为“未选择”,呈现本页。语言状态口径只引用 `flows/scoop-gui.md` §2.5,启动阶段引用 §2.6。

### 3.2 内部流转

展示中文与 English 两种语言预览 → 用户选择其中一种 → 用户确认 → 选择立即生效并写入本机 → 启动序列继续进入“检测 Scoop”。

### 3.3 出口

- 用户确认语言后,按 `flows/scoop-gui.md` §1.1 步骤 3/4 与 §2.6.3-2 进入检测 Scoop 阶段,由 P01 承接检测或协助安装。
- 若后续检测结果为已检测到 Scoop,按 `flows/scoop-gui.md` §2.6.3-3/8 进入 P03。
- 后续再次启动时,若语言偏好已存在,按 `flows/scoop-gui.md` §1.1 步骤 2 不再进入本页。

## 4. 交互说明

- 场景 1:首次无语言偏好
  - 触发:用户首次启动,本机未保存界面语言选择。
  - 呈现:界面只展示“中文”和“English”两个选项及对应预览。
  - 结果:用户必须选择其一后才能继续启动流程。
- 场景 2:选择中文
  - 触发:用户选择“中文”并确认。
  - 呈现:当前页面及后续启动流程立即使用中文文案。
  - 结果:语言选择保存到本机,启动流程进入检测 Scoop。
- 场景 3:选择 English
  - 触发:用户选择“English”并确认。
  - 呈现:当前页面及后续启动流程立即使用英文文案。
  - 结果:语言选择保存到本机,启动流程进入检测 Scoop。
- 场景 4:保存失败
  - 触发:用户确认语言后,本机保存失败。
  - 呈现:界面告知本次选择已即时生效,但下次启动可能仍需重新选择。
  - 结果:启动流程继续推进,不新增第三种语言状态。

## 5. 验收要点

- ✅ 本机没有持久化语言偏好时,首次启动应展示 P02。
- ✅ 本机已有持久化语言偏好时,启动不应展示 P02。
- ✅ 页面语言选项只能是“中文”和“English”。
- ✅ 页面不应出现第三方语言、自动翻译入口或语言包导入入口。
- ✅ 用户未选择语言时,不应继续进入检测 Scoop 阶段。
- ✅ 用户确认“中文”后,后续界面应立即使用中文文案。
- ✅ 用户确认“English”后,后续界面应立即使用英文文案。
- ✅ 用户完成选择后,后续启动应直接使用上次选择,不再展示本页。
- ✅ 用户完成选择后,启动流程应继续进入检测 Scoop 阶段,而不是停留在语言选择页。
- ✅ 语言选择保存失败时,用户应看到本次即时生效与下次启动可能回退的说明。
- ✅ 本页的所有面向用户文案应同时具备中文与英文两套,不得半中半英。
