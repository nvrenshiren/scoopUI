# P04 · 已装软件包列表页

## 1. 目的

展示本机已安装的软件包及其版本信息,并清晰标注存在更新版本的过期项。用户可从本页进入详情、卸载、单包更新与批量更新所有过期软件包流程。

## 2. 功能清单

- 功能 1:F01 软件包浏览 —— 展示本机已装软件包列表。
- 功能 2:F03 软件包状态查询(过期检查) —— 对已装软件包展示是否存在过期标记。
- 功能 3:F04 软件包详情查看 —— 用户选择单个已装软件包后进入 P07 查看详情。
- 功能 4:F06 软件包卸载(单包) —— 用户可对单个已装软件包发起卸载。
- 功能 5:F07 软件包更新(单包) —— 用户可对单个过期软件包发起更新。
- 功能 6:F08 批量更新所有过期软件包 —— 用户可一次性更新当前所有过期软件包。
- 功能 7:F17 安装任务进度反馈 —— 卸载、更新、批量更新进入 P08 展示进度与结果。
- 功能 8:F19 错误处理与友好提示 —— 列表为空、读取失败、操作失败时给出下一步动作。

## 3. 页面流转

### 3.1 入口

本页面从 P03 主界面壳进入,属于 `flows/scoop-gui.md` §1.2 日常主界面流程中的已装软件包浏览区段。用户从卸载、单包更新或批量更新流程返回时,分别引用 `flows/scoop-gui.md` §1.4、§1.5、§1.6 的业务结果刷新本页。

### 3.2 内部流转

已装列表展示 → 用户在当前列表内过滤或定位 → 选中条目 → 进入 P07 查看详情或直接发起卸载 / 更新 → 写操作进入 P08 → 完成后刷新本页。应用包状态口径只引用 `flows/scoop-gui.md` §2.1,长时任务口径只引用 §2.4。

### 3.3 出口

- 选中软件包查看详情时进入 P07,对应 `flows/scoop-gui.md` §1.2 中 C04。
- 发起卸载时进入 P08,对应 `flows/scoop-gui.md` §1.4。
- 发起单包更新时进入 P08,对应 `flows/scoop-gui.md` §1.5。
- 发起批量更新所有过期软件包时进入 P08,对应 `flows/scoop-gui.md` §1.6。
- 列表为空或读取失败时在本页内呈现 P10 子状态,引用 `flows/scoop-gui.md` §1.9。

## 4. 交互说明

- 场景 1:查看已装列表
  - 触发:用户从主界面导航进入本页。
  - 呈现:界面展示本机已装软件包、版本、所属来源与过期标记。
  - 结果:用户可以判断哪些软件包可查看、可卸载、可更新。
- 场景 2:筛选已装列表
  - 触发:用户在当前已装列表中输入关键字或选择过滤条件。
  - 呈现:界面只保留符合条件的已装软件包。
  - 结果:用户可以快速定位目标项。
- 场景 3:查看详情
  - 触发:用户选中某个已装软件包。
  - 呈现:打开 P07,展示该软件包详情与可用操作。
  - 结果:用户可关闭返回本页,或继续发起写操作。
- 场景 4:单包卸载
  - 触发:用户对某个已装软件包选择卸载并确认。
  - 呈现:进入 P08 展示卸载过程与结果。
  - 结果:成功后该软件包不再出现在已装列表;失败时保留原有可见状态并允许重试。
- 场景 5:单包更新
  - 触发:用户对过期软件包选择更新并确认。
  - 呈现:进入 P08 展示更新过程与结果。
  - 结果:成功后该软件包的过期标记消失;失败时仍保留过期标记并允许重试。
- 场景 6:批量更新所有
  - 触发:用户选择批量更新所有过期软件包并确认对象清单。
  - 呈现:进入 P08 展示每个对象的执行进度与汇总结果。
  - 结果:成功项过期标记消失,失败项仍可单独重试。

## 5. 验收要点

- ✅ 进入本页时,用户应看到本机已装软件包列表,而不是可装软件包列表。
- ✅ 每个已装软件包应展示足以识别该软件包的名称与版本信息。
- ✅ 存在更新版本的软件包应有可见的过期标记。
- ✅ 未过期的软件包不应展示为可更新对象。
- ✅ 批量更新所有只应作用于当前被判定为过期的软件包。
- ✅ 当前没有过期软件包时,用户不应被引导执行批量更新所有。
- ✅ 用户发起卸载、单包更新或批量更新后,应看到 P08 中可观察的进度与最终结果。
- ✅ 卸载成功后,该软件包应从已装列表中消失;若仍属于已添加桶范围,应能在可装列表中重新出现。
- ✅ 单包更新成功后,该软件包的过期标记应消失。
- ✅ 批量更新完成后,成功项与失败项应能被用户区分。
- ✅ 操作失败时,用户应看到失败原因与重试入口,而不是只看到空白或关闭提示。
- ✅ 用户取消长时操作时,对应软件包应按操作前的业务状态继续呈现,且不应被标记为失败。
- ✅ 已装列表为空时,本页应显示 P10 空状态,并提供明确下一步动作。
- ✅ 列表读取失败时,本页应显示 P10 错误状态,并允许用户重试。
