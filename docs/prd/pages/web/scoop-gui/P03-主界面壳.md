## 1. 目的

作为产品进入主流程后的统一入口,承载侧边导航、主内容区与设置入口。该页面让用户能在已装软件包、可装软件包、桶管理与语言设置之间切换,并保持主界面使用连续性。

## 2. 功能清单

- 功能 1:F01 软件包浏览 —— 通过主区入口承载已装软件包列表与可装软件包列表。
- 功能 2:F09 已添加桶列表查看 —— 通过导航入口进入桶管理页查看本机已添加桶。
- 功能 3:F14 界面语言切换(产品内) —— 提供进入 P09 的设置入口。
- 功能 4:F18 单实例约束 —— 再次启动产品时聚焦已运行实例,不创建并发主界面。
- 功能 5:F19 错误处理与友好提示 —— 当主区读取失败或运行期间 Scoop 整体不可用时,承载友好提示入口。

## 3. 页面流转

### 3.1 入口

本页面从 `flows/scoop-gui.md` §1.2 日常主界面流程进入;前置条件是 Boot Sequence 已按 `flows/scoop-gui.md` §2.6.3-8 进入“进入主界面”。Scoop 自身是否可用的状态口径只引用 `flows/scoop-gui.md` §2.2。

### 3.2 内部流转

侧边导航选择 P04 / P05 / P06 → 主区显示对应页面 → 用户可通过设置入口进入 P09 → 再次返回时保留主区导航上下文。若主区出现空数据或读取失败,以 P10 子状态呈现。

### 3.3 出口

- 导航到 P04 已装软件包列表页,对应 `flows/scoop-gui.md` §1.2 中 C01 / C03 / C04 的浏览入口。
- 导航到 P05 浏览与搜索软件包页,对应 `flows/scoop-gui.md` §1.2 中 C01 / C02 的浏览与搜索入口。
- 导航到 P06 桶管理页,对应 `flows/scoop-gui.md` §1.2 与 §1.7 的桶管理入口。
- 导航到 P09 设置与语言切换页,对应 `flows/scoop-gui.md` §1.8 语言切换流程。

## 4. 交互说明

- 场景 1:进入主界面
  - 触发:启动序列确认语言已选择且 Scoop 已可用。
  - 呈现:界面展示主导航与默认主区内容。
  - 结果:用户可以开始浏览或进入其它业务页面。
- 场景 2:切换导航
  - 触发:用户在侧边导航中选择已装软件包、浏览搜索或桶管理。
  - 呈现:主区切换到对应页面,导航选中状态同步变化。
  - 结果:用户在不同业务页间切换时,已输入的筛选或搜索上下文不应无故丢失。
- 场景 3:进入设置
  - 触发:用户从主界面设置入口进入语言设置。
  - 呈现:主区或设置区域展示 P09。
  - 结果:用户完成语言切换后返回主界面,所有文案按新语言显示。
- 场景 4:再次启动产品
  - 触发:本机已有运行实例时,用户再次启动产品。
  - 呈现:已有实例被唤起并聚焦到用户当前使用上下文。
  - 结果:不出现第二个可独立操作的主界面。
- 场景 5:运行期间无法读取
  - 触发:进入主界面后,某个主区数据无法读取。
  - 呈现:对应主区显示 P10 友好提示,不打断其它可用区段。
  - 结果:用户获得重试或返回其它页面的明确下一步动作。

## 5. 验收要点

- ✅ 只有在 Boot Sequence 进入“进入主界面”后,用户才应看到 P03。
- ✅ 未检测到 Scoop 或协助安装未成功时,用户不应进入 P03。
- ✅ 主界面应提供进入 P04、P05、P06、P09 的清晰入口。
- ✅ 用户在 P04 / P05 / P06 之间切换时,各页面已输入的搜索或筛选条件不应无故丢失。
- ✅ 用户再次启动产品时,应聚焦已有实例,不应出现第二个可独立操作的实例。
- ✅ 被唤起的已有实例应保持用户此前所在页面或可理解地回到主界面。
- ✅ 用户从设置页完成语言切换后,主界面导航与主区文案应立即按新语言显示。
- ✅ 运行期间某个主区读取失败时,主界面应显示友好错误提示,而不是空白或崩溃。
- ✅ 运行期间 Scoop 整体不可用时,主界面应提示无法读取,但不应自动进入首次协助安装流程。
- ✅ 主界面不应提供登录、角色切换、多用户管理等不属于本阶段的入口。
