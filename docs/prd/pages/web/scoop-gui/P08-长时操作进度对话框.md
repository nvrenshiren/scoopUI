# P08 · 长时操作进度对话框

> 文档层级:页面级契约
> 适用范围:本页面(electron 端 scoop-gui 模块)

## 1. 目的

统一展示安装、卸载、更新、桶添加、桶移除、协助安装 Scoop 等长时操作的进度、结果与失败重试入口。该对话框让用户在等待期间持续了解当前动作是否仍在推进、是否成功、是否需要处理失败。

## 2. 功能清单

- 功能 1:F17 安装任务进度反馈 —— 展示 F05 / F06 / F07 / F08 / F11 / F12 / F16 产生的长时任务进度、结果与终止状态。
- 功能 2:F19 错误处理与友好提示 —— 对失败、取消、网络失败、命令级失败等异常结果给出业务语言说明与重试入口。

## 3. 页面流转

### 3.1 入口

本页面由 P01 / P04 / P05 / P06 / P07 触发长时动作时进入。对应流程包括 `flows/scoop-gui.md` §1.1 协助安装、§1.3 单包安装、§1.4 单包卸载、§1.5 单包更新、§1.6 批量更新所有过期软件包、§1.7 桶管理流程。长时任务状态口径只引用 `flows/scoop-gui.md` §2.4。

### 3.2 内部流转

任务创建 → 进度文本持续追加或更新 → 展示当前对象与动作类型 → 任务完成后展示成功 / 失败 / 取消结果 → 成功时允许关闭并刷新来源页 → 失败时允许重试或关闭返回来源页。涉及应用包、桶、Scoop 自身的状态变化分别只引用 `flows/scoop-gui.md` §2.1、§2.3、§2.2。

### 3.3 出口

- 从 P01 进入的协助安装成功后,按 `flows/scoop-gui.md` §1.1 与 §2.6.3-6/8 进入 P03;失败或取消后返回 P01。
- 从 P04 进入的卸载、单包更新或批量更新完成后,返回 P04 并刷新已装列表,对应 `flows/scoop-gui.md` §1.4 / §1.5 / §1.6。
- 从 P05 或 P07 进入的安装完成后,返回来源页并刷新软件包状态,对应 `flows/scoop-gui.md` §1.3。
- 从 P06 进入的添加或移除桶完成后,返回 P06 并刷新桶列表与可装范围,对应 `flows/scoop-gui.md` §1.7。

## 4. 交互说明

- 场景 1:展示执行进度
  - 触发:用户确认发起任一长时动作。
  - 呈现:对话框展示动作类型、目标对象、当前阶段与持续更新的进度文本。
  - 结果:用户能够判断任务仍在排队、执行或已结束。
- 场景 2:成功完成
  - 触发:任务达到业务目标。
  - 呈现:对话框展示成功结果与关闭入口。
  - 结果:用户关闭后返回来源页,来源页刷新为最新业务状态。
- 场景 3:失败可重试
  - 触发:任务未达成业务目标且允许重试。
  - 呈现:对话框展示失败原因、目标对象、重试入口与关闭入口。
  - 结果:用户可立即重试同类动作,也可返回来源页处理其它事项。
- 场景 4:用户取消
  - 触发:用户在任务排队或执行期间主动取消。
  - 呈现:对话框展示取消结果,并说明该动作未按失败处理。
  - 结果:用户返回来源页后,目标对象保持动作前的业务状态。
- 场景 5:批量更新
  - 触发:用户从 P04 发起批量更新所有过期软件包。
  - 呈现:对话框展示本次对象清单、逐项进度与最终汇总。
  - 结果:用户能区分成功项、失败项与被取消项。
- 场景 6:多个任务
  - 触发:用户在产品运行期间先后发起多个长时任务。
  - 呈现:每个任务的目标、进度与结果可被区分,不混淆文本或结果。
  - 结果:用户能准确判断每个任务的最终状态。

## 5. 验收要点

- ✅ 用户发起安装、卸载、更新、批量更新、添加桶、移除桶或协助安装 Scoop 后,应看到 P08。
- ✅ P08 应展示当前动作类型与目标对象,用户能判断正在处理什么。
- ✅ 长时动作执行期间,用户应看到可观察的进度变化,而不是静止空白。
- ✅ 任务成功时,用户应看到明确的成功结果。
- ✅ 任务失败时,用户应看到失败原因与重试入口。
- ✅ 任务取消时,用户应看到取消结果,且取消不应被误报为失败。
- ✅ 成功关闭 P08 后,来源页应刷新并体现最新业务状态。
- ✅ 从 P01 发起的协助安装成功后,应进入 P03;失败或取消后应回到 P01。
- ✅ 从 P04 发起的更新或卸载完成后,应回到 P04 并刷新已装列表。
- ✅ 从 P05 或 P07 发起的安装完成后,来源页应反映该软件包状态变化。
- ✅ 从 P06 发起的桶添加或移除完成后,应刷新已添加桶与已知桶相关呈现。
- ✅ 批量更新时,应展示逐项结果与汇总结果,失败项不应掩盖成功项。
- ✅ 单项失败不应阻止批量更新中的后续项继续完成。
- ✅ 多个长时任务的进度与结果不应互相串扰或覆盖。
- ✅ 用户重试失败任务时,应能看到新一轮进度,而不是复用旧结果误导用户。
