# P06 · 桶管理页

## 1. 目的

帮助用户查看本机已添加桶与 Scoop 已知桶清单,并完成添加桶、移除桶等管理动作。该页面让可装软件包来源范围的变化对用户可见、可控、可恢复。

## 2. 功能清单

- 功能 1:F09 已添加桶列表查看 —— 展示当前本机已添加桶。
- 功能 2:F10 已知桶清单查看 —— 展示 Scoop 自身维护的可供添加的桶清单。
- 功能 3:F11 桶添加 —— 用户可从已知桶清单中选择一个未添加桶并发起添加。
- 功能 4:F12 桶移除 —— 用户可从已添加桶列表中选择一个桶并发起移除。
- 功能 5:F17 安装任务进度反馈 —— 添加桶、移除桶进入 P08 展示进度与结果。
- 功能 6:F19 错误处理与友好提示 —— 列表为空、读取失败、添加失败、移除失败时给出友好提示与重试入口。

## 3. 页面流转

### 3.1 入口

本页面从 P03 主界面壳进入,属于 `flows/scoop-gui.md` §1.2 日常主界面流程中的桶清单查看入口。添加与移除动作进入 `flows/scoop-gui.md` §1.7 桶管理流程,其中添加引用 §1.7.1,移除引用 §1.7.2。

### 3.2 内部流转

已添加 Tab / 已知 Tab 切换 → 用户在已知 Tab 选择未添加桶并确认添加 → 进入 P08 → 成功后刷新两个 Tab;用户在已添加 Tab 选择目标桶并确认移除 → 进入 P08 → 成功后刷新两个 Tab。桶状态口径只引用 `flows/scoop-gui.md` §2.3,长时任务口径只引用 §2.4。

### 3.3 出口

- 从已知 Tab 发起添加桶时进入 P08,对应 `flows/scoop-gui.md` §1.7.1。
- 从已添加 Tab 发起移除桶时进入 P08,对应 `flows/scoop-gui.md` §1.7.2。
- 添加或移除完成后关闭 P08 回到本页,并按 `flows/scoop-gui.md` §1.7 业务结果刷新桶列表与可装软件包范围。
- 列表为空或读取失败时在本页内呈现 P10 子状态,引用 `flows/scoop-gui.md` §1.9。

## 4. 交互说明

- 场景 1:查看已添加桶
  - 触发:用户进入本页或切换到已添加 Tab。
  - 呈现:界面展示本机当前已添加桶列表。
  - 结果:用户可选择其中一个桶发起移除。
- 场景 2:查看已知桶
  - 触发:用户切换到已知 Tab。
  - 呈现:界面展示可供添加的已知桶清单,并区分当前已添加与未添加的业务状态。
  - 结果:用户可从未添加桶中选择目标进行添加。
- 场景 3:添加桶
  - 触发:用户在已知 Tab 选择未添加桶并确认添加。
  - 呈现:进入 P08 展示添加过程与结果。
  - 结果:成功后该桶出现在已添加 Tab,已知 Tab 不再把它作为可添加对象展示。
- 场景 4:移除桶
  - 触发:用户在已添加 Tab 选择目标桶并确认移除。
  - 呈现:进入 P08 展示移除过程与结果。
  - 结果:成功后该桶不再出现在已添加 Tab,原本属于该桶的可装软件包不再作为可装范围展示。
- 场景 5:添加或移除失败
  - 触发:P08 返回添加失败或移除失败结果。
  - 呈现:本页保留失败对象的业务状态,并提供重试或放弃入口。
  - 结果:用户可再次发起同类动作,或回到列表继续其它操作。

## 5. 验收要点

- ✅ 本页应提供“已添加”和“已知”两个清晰区段或切换入口。
- ✅ 已添加区段应展示本机当前已添加桶。
- ✅ 已知区段应展示 Scoop 自身维护的已知桶清单。
- ✅ 已知区段中已添加的桶不应再作为可添加对象展示。
- ✅ 用户从已知区段成功添加一个桶后,该桶应出现在已添加区段。
- ✅ 用户从已知区段成功添加一个桶后,回到已知区段时该桶不应继续作为可添加对象出现。
- ✅ 用户从已添加区段成功移除一个桶后,该桶不应继续出现在已添加区段。
- ✅ 用户成功移除桶后,原本只属于该桶的可装软件包不应继续出现在 P05 的可装范围中。
- ✅ 添加桶失败时,用户应看到失败原因与重试入口。
- ✅ 移除桶失败时,用户应看到失败原因与重试入口。
- ✅ 用户取消添加或移除时,对应桶应回到动作前的业务状态,不应被标记为失败。
- ✅ 添加或移除过程中,用户应在 P08 看到可观察的进度与最终结果。
- ✅ 已添加桶列表为空时,应显示 P10 空状态并引导用户前往已知区段添加桶。
- ✅ 已知桶清单读取失败时,应显示 P10 错误状态并允许用户重试。
