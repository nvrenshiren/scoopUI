# P07 · 软件包详情对话框

## 1. 目的

集中展示单个软件包的业务详情,并根据该软件包当前状态提供安装、卸载或更新入口。该对话框帮助用户在执行写操作前确认目标对象与可用动作。

## 2. 功能清单

- 功能 1:F04 软件包详情查看 —— 展示单个软件包的名称、版本、所属桶、说明等业务信息。
- 功能 2:F05 软件包安装(单包) —— 当软件包为可装未装时,提供安装入口。
- 功能 3:F06 软件包卸载(单包) —— 当软件包为已装时,提供卸载入口。
- 功能 4:F07 软件包更新(单包) —— 当软件包为过期时,提供更新入口。
- 功能 5:F17 安装任务进度反馈 —— 安装、卸载、更新进入 P08 展示进度与结果。
- 功能 6:F19 错误处理与友好提示 —— 详情读取失败或操作失败时给出友好提示与下一步动作。

## 3. 页面流转

### 3.1 入口

本页面从 P04 已装软件包列表页或 P05 浏览与搜索软件包页进入,对应 `flows/scoop-gui.md` §1.2 日常主界面流程中的 C04 查看详情入口。若用户在详情内发起安装、卸载或更新,分别进入 `flows/scoop-gui.md` §1.3、§1.4、§1.5。

### 3.2 内部流转

详情展示 → 根据当前软件包状态呈现可用操作 → 用户确认安装 / 卸载 / 更新 → 进入 P08 → 完成后关闭或返回来源页并刷新。应用包状态口径只引用 `flows/scoop-gui.md` §2.1,长时任务口径只引用 §2.4。

### 3.3 出口

- 用户关闭详情时,返回来源页 P04 或 P05,不改变软件包业务状态。
- 用户触发安装时进入 P08,对应 `flows/scoop-gui.md` §1.3。
- 用户触发卸载时进入 P08,对应 `flows/scoop-gui.md` §1.4。
- 用户触发更新时进入 P08,对应 `flows/scoop-gui.md` §1.5。
- 操作完成后回到来源页,来源页按对应流程业务结果刷新。

## 4. 交互说明

- 场景 1:查看已装软件包详情
  - 触发:用户在 P04 选择一个已装软件包。
  - 呈现:对话框展示该软件包的详情,并提供卸载入口;若该软件包过期,同时提供更新入口。
  - 结果:用户可关闭返回 P04,或确认执行卸载 / 更新。
- 场景 2:查看可装软件包详情
  - 触发:用户在 P05 选择一个可装软件包。
  - 呈现:对话框展示该软件包的详情,并提供安装入口。
  - 结果:用户可关闭返回 P05,或确认执行安装。
- 场景 3:过期软件包更新
  - 触发:用户在详情内对过期软件包选择更新。
  - 呈现:进入 P08 展示更新过程与结果。
  - 结果:成功后过期标记消失,来源页刷新。
- 场景 4:操作不可用
  - 触发:软件包当前状态不支持某项动作。
  - 呈现:对话框不展示该动作,或以业务语言说明不可操作原因。
  - 结果:用户不会对错误状态的软件包发起不成立的动作。
- 场景 5:详情读取失败
  - 触发:用户打开详情时,详情信息无法读取。
  - 呈现:对话框展示友好错误提示与重试 / 关闭入口。
  - 结果:用户可重试读取或回到来源页。

## 5. 验收要点

- ✅ 用户从 P04 选择已装软件包时,应打开该软件包的详情对话框。
- ✅ 用户从 P05 选择可装软件包时,应打开该软件包的详情对话框。
- ✅ 详情内容应来源于 Scoop 所识别的软件包信息,不得展示产品自造的软件源信息。
- ✅ 可装未装软件包详情中应提供安装入口,不应提供卸载入口。
- ✅ 已装软件包详情中应提供卸载入口,不应把该软件包当作未装软件重复安装。
- ✅ 已装且过期的软件包详情中应提供更新入口。
- ✅ 已装且最新版的软件包详情中不应提供更新入口。
- ✅ 用户触发安装、卸载或更新后,应进入 P08 查看进度与最终结果。
- ✅ 用户关闭详情对话框时,应返回来源页并保持来源页上下文。
- ✅ 操作成功后,来源页应刷新并体现软件包业务状态变化。
- ✅ 操作失败后,用户应看到失败原因与重试入口。
- ✅ 详情读取失败时,用户应看到可理解的错误提示,并可选择重试或关闭。
