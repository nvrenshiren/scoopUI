# P01 · 启动检测与协助安装页

> 文档层级:页面级契约
> 适用范围:本页面(electron 端 scoop-gui 模块)

## 1. 目的

在首次启动或启动检测发现 Scoop 不可用时,向用户清晰呈现检测结果,并引导用户在产品内完成 Scoop 的协助安装。该页面解决用户不知道为何无法进入主界面、以及如何完成首次安装的问题。

## 2. 功能清单

- 功能 1:F15 启动时 Scoop 可用性检测 —— 在启动序列中展示当前是否检测到可用的 Scoop。
- 功能 2:F16 协助安装 Scoop —— 当未检测到 Scoop 时,允许用户填写 `ScoopDir` / `ScoopGlobalDir` / `ScoopCacheDir` / `NoProxy` / `Proxy` / `ProxyCredential` / `ProxyUseDefaultCredentials` / `RunAsAdmin` 八项安装配置并确认安装。
- 功能 3:F17 安装任务进度反馈 —— 用户确认安装后,进入 P08 展示安装过程、结果与失败重试入口。
- 功能 4:F19 错误处理与友好提示 —— 对检测失败、安装失败、取消等情形给出业务语言提示与下一步动作。

## 3. 页面流转

### 3.1 入口

本页面从 `flows/scoop-gui.md` §1.1 首次启动流程进入:启动序列完成语言偏好确认后进入“检测 Scoop”;若检测结果为“未检测到 Scoop”,停留在本页面并提供协助安装入口。对应启动阶段引用 `flows/scoop-gui.md` §2.6 Boot Sequence, Scoop 自身状态口径引用 §2.2。

### 3.2 内部流转

检测状态展示 → 未检测到 Scoop 时展示安装说明 → 用户填写八项安装配置 → 页面校验配置完整性与风险提示 → 用户确认 → 进入 P08 长时操作进度对话框。安装任务的状态口径只引用 `flows/scoop-gui.md` §2.4 InstallJob,不在本页复述。

### 3.3 出口

- 安装成功并复检通过后,按 `flows/scoop-gui.md` §1.1 步骤 7 与 §2.6.3-6/8 进入 P03 主界面壳。
- 安装失败或用户取消后,按 `flows/scoop-gui.md` §1.1 异常分支与 §2.6.3-7 回到本页面,允许用户修改配置后重试。
- 若检测直接判定 Scoop 已可用,按 `flows/scoop-gui.md` §2.6.3-3/8 不停留本页,进入 P03。

## 4. 交互说明

- 场景 1:启动检测中
  - 触发:用户完成首次语言选择或已有语言偏好后继续启动。
  - 呈现:界面显示正在检测 Scoop 是否可用,并避免用户误以为应用无响应。
  - 结果:检测完成后进入主界面或进入协助安装状态。
- 场景 2:未检测到 Scoop
  - 触发:启动检测结果为未检测到 Scoop。
  - 呈现:界面说明当前无法进入主界面的原因,并展示协助安装入口。
  - 结果:用户可以开始填写安装配置,也可以停留在本页不继续。
- 场景 3:填写安装配置
  - 触发:用户选择协助安装。
  - 呈现:界面展示八项配置项、默认值、业务说明与必要的权限提示。
  - 结果:配置通过校验后允许用户一次性确认安装。
- 场景 4:涉及全局目录
  - 触发:用户填写或启用 `ScoopGlobalDir` 等需要管理员权限的配置。
  - 呈现:界面提示该选择可能需要管理员权限,并说明不选择管理员权限时的影响。
  - 结果:用户在明确风险后继续确认或返回修改。
- 场景 5:安装失败后重试
  - 触发:P08 返回安装失败结果。
  - 呈现:本页保留用户可理解的失败原因与上次配置,并提供重试或修改配置的路径。
  - 结果:用户可再次进入 P08 发起新一轮协助安装。

## 5. 验收要点

- ✅ 未持久化语言偏好的首次启动用户,应先看到 P02;语言确认后才进入本页的检测阶段。
- ✅ 已完成语言选择且未检测到 Scoop 的用户,启动后应看到本页,而不是直接进入 P03。
- ✅ 检测进行中时,用户应能看到明确的检测状态提示。
- ✅ 检测到 Scoop 已可用时,用户不应被要求填写安装配置,应进入 P03。
- ✅ 未检测到 Scoop 时,页面应说明无法进入主界面的业务原因,并提供协助安装入口。
- ✅ 页面应只提供 `ScoopDir` / `ScoopGlobalDir` / `ScoopCacheDir` / `NoProxy` / `Proxy` / `ProxyCredential` / `ProxyUseDefaultCredentials` / `RunAsAdmin` 八项安装配置,不得出现额外安装选项。
- ✅ 八项安装配置的默认值应与官方 `install.ps1` 默认值一致。
- ✅ 用户填写无效路径、冲突代理设置或不完整代理凭据时,应在确认安装前看到可理解的校验提示。
- ✅ 用户选择 `ScoopGlobalDir` 等需要管理员权限的配置时,应看到管理员权限提示,但不应被当作另一类用户角色处理。
- ✅ 用户确认安装后,应进入 P08 并看到安装进度可观察地推进。
- ✅ 安装过程中,用户不应再修改本次已确认的安装配置。
- ✅ 安装失败时,用户应看到失败原因、重试入口与修改配置入口。
- ✅ 安装成功后,页面应完成复检并进入 P03;若复检未通过,不得进入 P03。
- ✅ 用户取消协助安装后,应停留在无法进入主界面的路径上,并可再次触发协助安装。
- ✅ 确认后的安装配置应在本机保留,后续重新安装场景可复用。
