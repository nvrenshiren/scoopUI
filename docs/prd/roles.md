## 2. 能力清单

以下能力均来自 `project.md` 第 3.1 节"本阶段做",由本机用户在本产品中触发。系统侧的"自动检测"动作不视为独立的用户能力,仅作为某项能力的伴随行为。

| C06 | 卸载软件包 | 对单个已装软件包执行卸载 |
| C07 | 单包更新 | 对单个已装软件包升级到当前桶内的最新版本 |
| C08 | 批量更新所有过期软件包 | 一次性更新本机所有处于过期状态的软件包 |
| C09 | 查看已添加桶列表 | 查看当前已经添加到本机的桶列表 |

> - C16 的"填写配置项 / 执行安装 / 查看进度 / 失败重试"按 project.md 3.1 描述合并为一项业务能力,不拆分。

## 4. 约束说明

以下条目均为"受条件约束",逐项展开为业务语句。

### 4.1 C13 选择界面语言(首次启动)

- **触发条件**:仅在用户**首次启动**本产品且本机尚未持久化界面语言选择时,允许用户进行首次界面语言选择。
- **持久化**:选择结果在本机持久化;后续启动按用户上次选择显示,不再询问。
- **可选值**:仅可在"中文"与"English"两套文案之间二选一,不允许选择其它语言。

### 4.2 C15 检测 Scoop 是否可用

- **触发条件**:产品启动进入主流程前自动执行,不需用户主动触发。
- **业务结果**:检测结果仅用于决定后续是否进入"协助安装 Scoop"(C16)流程;本机用户不直接对该检测行为做"允许 / 不允许"决策,但其结果是后续 C16 能否触发的依据。

### 4.3 C16 协助安装 Scoop

- **触发条件**:**仅当 C15 检测结果为"未检测到 Scoop"**(即本机 `scoop` 命令不可用)时,才允许用户触发 C16。
- **配置项范围**:用户可填写的安装配置项仅限 Scoop 官方安装脚本 `install.ps1` 已声明的参数集,包括安装根目录 `ScoopDir`、全局软件包目录 `ScoopGlobalDir`、下载缓存目录 `ScoopCacheDir`、代理设置 `NoProxy` / `Proxy`、代理凭据 `ProxyCredential` / `ProxyUseDefaultCredentials`、是否以管理员权限运行 `RunAsAdmin`;默认值采用 `install.ps1` 的默认值;不允许引入官方脚本未暴露的安装选项。
- **管理员权限**:当用户选择"全局软件包目录"(`ScoopGlobalDir`)等需要管理员权限的选项时,产品需提示该选项需要管理员权限并协助其提权执行,但**不强制要求管理员**;即用户可以填写该项,但实际执行是否需要提权以该项业务含义为准。
- **一次性确认**:所有配置项在执行安装前一次性确认,安装过程中不再允许改动。
- **持久化**:确认后的配置项在本机持久化,以便后续重新安装 Scoop 时复用。
- **结果与重试**:安装过程中提供可观察的进度与结果反馈;失败时允许重试,并以业务语言展示失败原因。
- **不允许触发**:当 C15 检测结果为"Scoop 已可用"时,**不允许**触发 C16;且本产品不对 Scoop 自身的卸载、升级、修复等维护操作负责。
