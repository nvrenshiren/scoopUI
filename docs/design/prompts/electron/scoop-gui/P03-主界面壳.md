## 1. 页面定位

P03 是产品完成 Boot Sequence 后进入日常主流程的 **桌面主界面壳**。它不是业务列表本身,而是承载 P04 / P05 / P06 / P09 的统一框架。

核心职责:

- 提供固定 240px Sidebar,包含 4 个导航项:已装 / 浏览 / 桶 / 设置。
- 提供 Content 主区,包含顶部导航(PageHeader)与主区 Outlet。
- 在导航切换时同步高亮当前项,同时保留各子页筛选/搜索上下文。
- 承载再次启动产品时已有实例被唤起并保持上下文的可理解反馈。
- 承载主区读取失败时的 P10 友好提示,不让界面空白或崩溃。

## 2. 整体布局(1280 × 800)

```
┌──────────────────────────── 1280 × 800 ────────────────────────────┐
│ Sidebar 240px                 │ Content 1040px                     │
│ ┌──────────────────────────┐  │ ┌────────────────────────────────┐ │
│ │ Logo scoop-gui           │  │ │ TopNav / PageHeader h-14       │ │
│ │ Version + status         │  │ │ Title + Breadcrumb + CmdK      │ │
│ │                          │  │ ├────────────────────────────────┤ │
│ │ 已装 Installed      ◀    │  │ │ PageBody p-6 overflow-auto     │ │
│ │ 浏览 Browse             │  │ │                                │ │
│ │ 桶 Buckets              │  │ │ [Outlet: P04/P05/P06/P09/P10]  │ │
│ │                          │  │ │                                │ │
│ │ flex-1 spacer            │  │ │                                │ │
│ │ 设置 Settings            │  │ │                                │ │
│ │ Theme summary            │  │ │                                │ │
│ └──────────────────────────┘  │ └────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

硬约束:

- 窗口原型必须为 `1280×800`。
- Sidebar 展开态固定 **240px**;本 P03 原型展示展开态,底部保留折叠入口但不展开折叠交互。
- Content 宽度为 `1040px`。
- 顶部导航 `h-14`(56px),主区 `744px` 高度内滚动。
- 当前选中导航项必须高亮,高亮背景使用 `--secondary`,左侧加 `--primary` 竖条增强可见性。

## 3. 关键组件

### 3.1 Sidebar(固定 240px)

结构从上到下:

1. 品牌区:
   - 32×32 图标容器,使用 `--primary` 背景 + lucide `Package` 图标。
   - 标题 `scoop-gui`,副标题显示当前 Scoop 状态,如"Scoop 可用 · v0.5.2"。
2. 主导航区:
   - 4 个导航项,高度均为 40px。
   - 图标全部来自 lucide-react,线性 1.5px stroke。
   - 不使用 emoji。
3. 底部区:
   - Settings 与 Theme summary 靠底。
   - Theme summary 仅显示当前主题策略:"跟随系统 / System"。
   - 折叠按钮作为视觉入口,实际折叠交互交由 developer。

导航项:

| 路由 | zh 文案 | en 文案 | 图标 | 目标页 |
| --- | --- | --- | --- | --- |
| installed | 已装 | Installed | `PackageCheck` | P04 |
| browse | 浏览 | Browse | `Search` | P05 |
| buckets | 桶 | Buckets | `Database` | P06 |
| settings | 设置 | Settings | `Settings` | P09 |

### 3.2 TopNav / PageHeader

顶部导航固定在 Content 顶部,高度 56px,边框底线 `--border`。

左侧:

- 当前页标题,`text-2xl font-semibold`。
- 面包屑,`text-xs text-muted-foreground`: `scoop-gui / installed`。

右侧:

- 全局搜索预留按钮:视觉为 `btn-outline h-10`,显示 `Cmd/Ctrl + K`。
- 状态徽标:正常为成功色点 + "Scoop 可用";错误场景为警告/危险色点。
- 语言即时切换后的标题与导航文案应同步变化。

### 3.3 PageBody / Outlet

PageBody 使用 `p-6`,内部根据当前导航显示子页占位:

- 已装:展示 P04 摘要占位,包括搜索框、过期统计、表格骨架。
- 浏览:展示 P05 摘要占位,包括搜索框、桶范围、可装列表。
- 桶:展示 P06 摘要占位,包括已添加桶/已知桶 tabs。
- 设置:展示 P09 摘要占位,重点展示语言切换入口。
- 错误:展示 P10 子状态,包含错误码、可读文案、重试/切换导航动作。

P03 不实现 P04/P05/P06/P09 的完整表格细节,只提供主壳与 Outlet 容器的真实尺寸、状态与交互上下文。

## 4. 状态切换矩阵(覆盖 PRD §4 全部场景)

| 场景 | PRD §4 | 触发 | Sidebar | TopNav | PageBody |
| --- | --- | --- | --- | --- | --- |
| 1. 进入主界面 | 场景 1 | Boot Sequence 确认语言已选且 Scoop 可用 | 默认高亮"已装" | 标题"已装软件包" | P04 摘要 + 最近上下文 |
| 2. 切换导航 | 场景 2 | 点击已装/浏览/桶 | 被点击项高亮 | 标题与面包屑同步 | 对应 Outlet 切换,保留其它页筛选/搜索摘要 |
| 3. 进入设置 | 场景 3 | 点击设置 | "设置"高亮 | 标题"设置" | P09 摘要,显示语言切换后全壳文案即时变化 |
| 4. 再次启动产品 | 场景 4 | 单实例锁命中 | 保持用户此前高亮项 | 保持此前标题 | 显示右下角 Toast:"已聚焦现有窗口",不创建第二主界面 |
| 5. 运行期间无法读取 | 场景 5 | 某主区读操作失败 | 当前业务项仍高亮 | 状态徽标转为 warning/destructive | 主区替换为 P10 友好提示,可重试或切换其它导航 |

关键交互:

- 导航切换不清空其它页的搜索、筛选或滚动上下文;可在 Sidebar 或 PageHeader 下方用小型摘要提示"已保留:搜索 7zip"。
- 设置页语言切换成功后,主壳所有文案立即切换,不需要重启。
- 再次启动只做聚焦反馈,不展示新的窗口框架或第二套主界面。
- 某个主区读取失败时,不能自动跳回首次协助安装流程。

## 5. 视觉决策(token 引用)

| 用途 | Token | 说明 |
| --- | --- | --- |
| 窗口背景 | `--background` | 整体基底 |
| Sidebar / Header | `--card` | 与主区同属工具壳 |
| 主文字 | `--foreground` | 标题、导航主文案 |
| 弱化文字 | `--muted-foreground` | 面包屑、副标题、描述 |

## 7. 交互细节

### 7.1 进入主界面

- Boot Sequence 进入"进入主界面"后才渲染 P03。
- 默认路由优先恢复用户上次停留页;若无记录,默认进入 P04 已装。
- Sidebar 与 PageHeader 在首帧同时出现,避免先白屏再跳转。

### 7.2 导航切换

- 点击 Sidebar 项后,当前项立即切换高亮。
- Content 只替换 PageBody Outlet,Sidebar 与 TopNav 容器不重建。
- 对 P04/P05/P06 保存 query/filter/tab 等本地上下文,返回时恢复。
- `Cmd/Ctrl + K` 是全局搜索入口预留,本原型只展示按钮与快捷键提示。

### 7.3 进入设置与语言切换

- Settings 是 Sidebar 第四项,与其它三项同级但位于底部区。
- 进入设置时 Sidebar 高亮"设置"。
- 用户在设置中切换语言后,Sidebar、TopNav、PageBody、Toast 文案立即切换。
- 不在 P03 里直接实现偏好写入细节,只定义视觉与行为。

### 7.4 再次启动产品

- 单实例锁命中时,已有窗口被唤起并聚焦。
- 不改变当前 Sidebar 高亮与主区上下文。
- 右下角显示 Toast 3-5 秒,说明"已聚焦现有窗口"。
- 不创建第二个窗口、不显示登录/用户切换入口。

### 7.5 主区读取失败

- 失败发生在哪个子页,哪个子页的 Sidebar 仍保持高亮。
- PageBody 替换为 P10 子状态,展示错误码 + 可读文案 + 下一步动作。
- 轻量错误可同时触发 Toast;主区严重不可读时必须有 P10 子状态。
- 不使用页面内联红条,不暴露原始堆栈。
- Scoop 整体不可用时,提示无法读取,但不自动回到 P01 首次协助安装。

## 9. 不在 P03 实现的元素(避免越界)

- ❌ P04 已装完整表格、批量更新按钮细节。
- ❌ P05 搜索完整结果与安装按钮状态机。
- ❌ P06 桶添加/移除完整流程。
- ❌ P07 详情对话框。
- ❌ P08 长时操作进度对话框。
- ❌ P09 偏好写入实现细节。
- ❌ 登录、账号、角色切换、多用户管理。
- ❌ 任何新的 IPC 通道、schema 或错误码定义。

## 10. 文档范围声明

本提示词只描述 P03 主界面壳的视觉结构、导航交互、场景覆盖与 i18n 文案 key。

不包含:

- React 组件文件结构 / 路由配置 / 状态管理实现。
- 具体 IPC 调用参数与返回结构。
- 子页面完整业务表格与对话框。
- 最终 i18n 文件落库位置。
- 任何绕过设计系统 token 的硬编码视觉规则。
