## 1. 页面定位

P08 是**通用进度对话框**,不是独立路由页。它由 P01(协助安装)、P04(卸载/单包更新/批量更新)、P05/P07(安装)、P06(桶添加/移除)发起任一长时动作时**自动弹出**,覆盖在来源页之上,展示动作类型、目标对象、当前阶段、原始日志与最终结果。

**该对话框的关键硬约束**(源自设计系统 v2 §8.5):

- **非 modal**:背景半透明 `bg-black/30 backdrop-blur-sm`,主页内容仍可见;但来源页交互禁用(§8.4)
- **必须可关闭**:右上角 X 按钮始终可见 + `aria-label="Close"` + Esc 键 + 底部 secondary "关闭"按钮
- **顶部 thin gradient bar**(`from-primary via-accent to-primary`)仅在 running 态显示,带 glow
- **进度条填充** `box-shadow: var(--glow-primary)`(科技感核心)
- **取消态颜色**:`--fg-muted` 中性灰,**禁用 destructive**(取消 ≠ 失败)
- **多个任务**:每个任务一个独立 P08 实例,级联偏移(§7.4)
- **字体**:`--font-heading` (Space Grotesk) / `--font-body` (DM Sans) / `--font-mono` (JetBrains Mono) 三套,严格不混用

## 3. 关键组件(引用设计系统 v2 §8.5 + §11)

### 3.1 半透明遮罩(non-modal overlay)

- 位置:`position: fixed; inset: 0; z-index: 40`
- 背景:`background-color: rgba(0, 0, 0, 0.30); backdrop-filter: blur(4px);`
- **不允许使用** 实色遮罩(那样就成 modal 了)
- 本身**不响应点击**(`pointer-events: none`),点击遮罩**不关闭**对话框

### 3.2 Dialog 容器

- 居中:`position: fixed; top: 50%; left: 50%; transform: translate(-50%, -50%);`
- 宽度:`max-width: 672px`(对应设计系统 `max-w-2xl`)
- 圆角:`--radius-xl`(16px)
- 阴影:`var(--shadow-xl)`
- 背景:`var(--bg-elevated)`
- 边框:`1px solid var(--border)`
- z-index:`z-index: 50`(在遮罩之上)

### 3.3 顶部 gradient bar(running 态)

- 高度 `2px`,宽度 100%
- `background: linear-gradient(to right, var(--primary), var(--accent), var(--primary))`
- `box-shadow: 0 0 12px rgba(34, 197, 94, 0.4)`(光晕)
- **仅 running 态显示**,终态(success/failed/cancelled)不显示(运行已结束)

### 3.4 DialogHeader

- `padding: 20px 24px 16px`,底边 `border-bottom: 1px solid var(--border)`
- **关闭按钮(X)**:绝对定位右上角,`40×40` ghost 按钮,lucide `X`,**始终可见**;`aria-label="Close"`,`title="Close (Esc)"`
- 标题区:左侧 [操作图标 36×36,rounded-full + soft 背景] + 中部 [DialogTitle `font-heading text-lg font-semibold` + 阶段描述 `font-body text-sm text-fg-muted`]

| 操作类型 | 图标(lucide) | soft 背景色 | 主色 |
| --- | --- | --- | --- |
| 安装 / 更新 / 桶添加 | `Download` / `RefreshCw` / `GitFork` | rgba(primary, 0.12) | `--primary` |
| 卸载 / 桶移除 | `Trash2` / `GitMerge` | rgba(fg-muted, 0.12) | `--fg-muted` |
| 终态-成功 | `CheckCircle2` | rgba(primary, 0.15) | `--primary` |
| 终态-失败 | `XCircle` | rgba(destructive, 0.15) | `--destructive` |
| 终态-取消 | `Ban` | rgba(fg-muted, 0.12) | `--fg-muted`(**非 destructive**) |

### 3.5 当前阶段 + 进度条(科技感核心)

- 阶段文本(对应 `ProgressChunk.stage`,API §2.4):
  - `downloading` → "下载中" / "Downloading"
  - `extracting` → "解压中" / "Extracting"
  - `installing` → "安装中" / "Installing"
  - `uninstalling` → "卸载中" / "Uninstalling"
  - `updating` → "更新中" / "Updating"
  - `cloning` → "克隆中" / "Cloning"
  - `removing` → "移除中" / "Removing"
  - `message` → "处理中" / "Processing"
- 文本格式:`font-body text-sm`,阶段词 `font-medium`,阶段标识 `font-mono text-xs text-fg-muted`,百分比 `font-mono text-sm font-medium`
- 进度条:
  - 容器 `height: 8px`(h-2),`border-radius: 9999px`,`background: var(--bg-overlay)`
  - 填充按状态:
    - running:`background: var(--primary); box-shadow: var(--glow-primary)`(科技感)
    - success:`background: var(--primary); box-shadow: var(--glow-primary)`
    - failed:`background: var(--destructive); box-shadow: var(--glow-destructive)`
    - **cancelled:`background: var(--fg-muted); box-shadow: none`**(中性灰,禁用 destructive)
  - indeterminate(无百分比):进度条内显示从左到右滑动的光条,1.5s 循环

### 3.6 原始日志区(v2 §8.5 硬约束)

- 高度固定 `height: 256px`(h-64)
- `font-family: var(--font-mono)`,`font-size: 12px`,`line-height: 1.6`
- 文字色 `var(--fg-muted)`
- 背景 `rgba(11, 18, 32, 0.5)`(bg-bg/50)
- 边框 `1px solid var(--border)`,圆角 `--radius-md`(8px)
- `overflow-y: auto`,`display: flex; flex-direction: column-reverse;` 实现"新行追加到底部并自动滚到底"
- 每行渲染 `ProgressChunk.raw`(API §2.4);阶段识别为 `message` 时整行加粗,其余原样展示
- 错误行 `.log-line.err` 用 `--destructive`,成功行 `.log-line.ok` 用 `--primary`
- **不允许对原始日志做翻译 / 美化**

### 3.7 DialogFooter

- `padding: 16px 24px`,顶边 `border-top: 1px solid var(--border)`
- 右对齐(主操作);左对齐 `jobId` font-mono 用于排错
- **四种状态对应不同按钮组**(PRD §4 场景 1-4):

| 状态 | 左下(jobId) | 次按钮 | 主按钮 | 视觉 |
| --- | --- | --- | --- | --- |
| **running** | font-mono text-fg-subtle | "后台运行" / "Run in background" btn-secondary | "停止" / "Stop" **btn-destructive**(强调停止) | glow-destructive |
| **success** | font-mono text-fg-subtle | "查看日志" / "View log" btn-secondary | "完成" / "Done" **btn-primary** | glow-primary |
| **failed** | font-mono text-fg-subtle | "关闭" / "Close" btn-secondary | "重试" / "Retry" **btn-primary**(鼓励继续) | glow-primary |
| **cancelled** | font-mono text-fg-subtle | — | "关闭" / "Close" **btn-secondary**(中性,**禁用 destructive**) | 无 glow |

按钮高度统一 `40px`(h-10);主按钮按 variant 应用对应 glow。

### 3.8 失败态错误展示(v2 §8.5 + §8.7)

- 错误码徽标:`.err-tag`,`font-mono text-[11px]`,`border: 1px solid var(--destructive); color: var(--destructive); background-color: rgba(239, 68, 68, 0.10);`
- 错误卡片:`background-color: rgba(239, 68, 68, 0.08); border: 1px solid rgba(239, 68, 68, 0.30); border-radius: var(--radius-lg);`
- 业务消息:`font-body text-sm text-fg`(由 `errors.<CODE>` i18n key 翻译,**不**展示 `cause` 原始堆栈)
- 失败对象明示:`font-body text-xs text-fg-muted` "目标:nodejs" 帮助用户识别失败项

### 3.9 取消态视觉(v2 §8.5 硬约束)

- 取消卡片:`background-color: rgba(161, 170, 184, 0.08); border: 1px solid rgba(161, 170, 184, 0.30);`(**非 destructive 红色**)
- 取消图标 `Ban` + 阶段进度 `--fg-muted` 中性灰
- 取消徽标:`.err-tag` 的 muted 变体,`border: 1px solid var(--fg-muted); color: var(--fg-muted); background-color: rgba(161, 170, 184, 0.10);`
- 文案突出"保持动作前的状态"(语义:取消 ≠ 失败)

### 3.10 批量项列表(场景 5)

仅在**批量更新**场景显示(`scoop:apps:update` 入参 `name` 省略):

- 综合进度条 + 7 行 app 列表
- 每行 `height: 40px`(h-10,§8.3 表格行高复用)
- 行内容:
  - 左:状态图标(`Check` 成功 / `Loader2 animate-spin` 进行中 / `XCircle` 失败 / 圆点 排队)
  - 中:app 名 `font-mono` + 版本过渡 `v1.2.3 → v1.2.4` (灰)
  - 右:状态标签 + 子进度条 / 错误码徽标 / "该项重试" 链接按钮
- 进行中行:`background-color: rgba(34, 197, 94, 0.05);` 弱化背景
- 失败行:`background-color: rgba(239, 68, 68, 0.05);` 弱化背景
- **失败项不掩盖成功项**(PRD 验收点 12-13):错误信息独立显示在对应行右,主汇总区仍准确统计成功/失败数

### 3.11 多任务并存(场景 6)

每个长时动作创建一个独立 Dialog 实例:

- **位置**:新对话框级联偏移(右下),`top: calc(50% + n*28px); left: calc(50% + n*28px)`
- **z-index**:新对话框置顶(后发起的覆盖先生成的)
- **互不串扰**:每个对话框独立订阅 `useProgress(jobId)`,日志/进度/状态各自独立
- 尺寸可缩小(`max-width: 560px`)以容纳并排展示

## 7. 交互细节

### 7.1 关闭机制(v2 §8.4 / §8.8)

| 触发方式 | running | 终态 |
| --- | --- | --- |
| 右上角 X | 弹"确认取消"二次确认 | 直接关闭 |
| Footer 取消/停止 | 触发取消(`E_JOB_CANCELLED`);子进程 kill | — |
| Esc 键 | 同 X 按钮 | 同 X 按钮 |
| Enter 键 | 同 Footer 主按钮 | 同 Footer 主按钮 |
| 点击遮罩 | **不响应**(pointer-events: none) | **不响应** |

### 7.2 日志区滚动行为(v2 §8.11)

- 新行追加到底部,自动 `scrollTop = scrollHeight`
- 用户向上滚动查看历史时,**暂停自动滚动**;用户再次滚到底部时恢复
- 实现:`flex flex-direction: column-reverse` + `IntersectionObserver` 监听底部 sentinel

### 7.3 长时操作期间来源页(v2 §8.4)

P08 打开时,所有其他页面的 Button / Input / 链接 / 表格行操作 `disabled` + `opacity-50 cursor-not-allowed`。视觉上仍可见(非 modal),但交互全部禁用。

### 7.4 多任务并存(场景 6 / PRD §5 验收点 8)

每个长时动作创建一个独立 Dialog 实例:

- **不共用对话框**:每个对话框绑定自己的 `jobId`
- **位置**:新对话框级联偏移(右下角),`top: calc(50% + n*28px); left: calc(50% + n*28px)`
- **z-index**:新对话框置顶
- **互不串扰**:每个对话框独立订阅 `useProgress(jobId)`

### 7.5 重试行为(对齐 PRD §5 验收点 14)

点击"重试"时:

- 对话框**不新建**实例,沿用同一 Dialog
- 进度条立即重置为 0%(indeterminate 动画过渡 200ms)
- 日志区**保留**(不清空),新一行追加 `--- Retrying at {ts} ---` 分隔
- Header 图标恢复为进行中态图标
- Footer 恢复为 `[后台运行] [停止]` 按钮组
