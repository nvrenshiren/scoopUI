## 1. 页面定位

P06 是 **P03 主界面壳内的列表页路由**(PRD §3.1,`flows/scoop-gui.md` §1.2 日常主界面流程),由 Sidebar 第 3 项"桶管理"导航进入。功能上对应 F09-F12、F17、F19,是 F05/F06 安装/卸载行为的"上游开关"(桶范围变化直接影响 P05 可装列表)。

因此:
- **不独立成窗**,嵌入 P03 主界面壳(设计系统 §9.1)
- Sidebar **保留**(240px),当前项 `桶管理` 高亮(`--secondary` 底)
- Content 区使用 P04/P05 通用列表骨架(设计系统 §9.2),通过 Tabs 区分两个区段

## 3. 关键组件(引用设计系统 §7)

### 3.1 PageHeader

- 高度 `h-14`、底部 `border-b`(设计系统 §9.2)
- 主标题:`text-2xl font-bold`(§3.2,P01/P03 同档)
  - 中:`桶管理` 英:`Buckets`
- 副文本(`text-sm text-muted-foreground`):
  - 中:`管理本机已添加的桶与 Scoop 已知桶清单。`
  - 英:`Manage locally added buckets and the known bucket catalog.`
- 右侧(可选)放"刷新"按钮(`ghost` variant,`lucide RefreshCw`,§8.10):触发 `scoop:buckets:list` + `scoop:buckets:known` 同时重读

### 3.2 Toolbar · Tabs(§7.9)

`TabsList` 高度 `h-10`,`TabsTrigger` 圆角 `rounded-md`,激活态:背景 `--background`、底部 2px `--primary` 边框。

| Tab | i18n key | 默认态 | 内容 | 触发查询 |
| --- | --- | --- | --- | --- |
| 已添加 | `buckets.manage.tab.added` | **默认激活**(PRD §4 场景 1) | 当前本机已添加桶列表 | `scoop:buckets:list` |
| 已知 | `buckets.manage.tab.known` | — | Scoop 维护的可添加桶清单 | `scoop:buckets:known` |

Tabs 右侧固定一个 `Btn 刷新 outline sm`(lucide `RefreshCw`,`h-8`),点击同时重读当前 Tab 与另一 Tab(因两 Tab 数据交叉:已知 Tab 的"已添加"判定依赖已添加列表)。

### 3.3 Table(已添加 Tab · §7.8)

行高 **40px(`h-10`)**,整行 `cursor-pointer`,点击行空白处打开"移除确认 AlertDialog"(避免点击按钮与点击行串扰,§8.3)。

| 列 | 字段(API §2.2 `BucketInfo`) | i18n key | 宽度策略 | 备注 |
| --- | --- | --- | --- | --- |
| 桶名 | `name` | `buckets.list.column.name` | flex-1,`font-medium` | 主列 |
| 仓库 | `source` | `buckets.list.column.source` | flex-2,`font-mono-c text-sm text-muted-fg` | URL,等宽 |
| 更新时间 | `updated` | `buckets.list.column.updated` | `w-32`,`text-sm text-muted-fg` | 原样字符串 |
| Manifest 数 | `manifests` | `buckets.list.column.manifests` | `w-24`,`text-sm tabular-nums` 右对齐 | 整数,缺失显示 `—` |
| 操作 | — | — | `w-24` 右对齐 | `[Btn 移除 destructive sm]` |

操作 cell 内放 `Btn 移除 destructive size=sm`(`h-8`,lucide `Trash2` + "移除"),点击直接打开 AlertDialog(行点击也开同一对话框,§8.3 串扰规避靠按钮 `e.stopPropagation()`)。

### 3.4 Table(已知 Tab · §7.8)

行高 `h-10`,整行可点击(点击行空白 → 打开"添加确认 AlertDialog";点击添加按钮 → 同一对话框)。

| 列 | 字段(`KnownBucket`) | i18n key | 宽度策略 | 备注 |
| --- | --- | --- | --- | --- |
| 桶名 | `name` | `buckets.list.column.name` | flex-1,`font-medium` | 主列 |
| 仓库 | `repo` | `buckets.list.column.source` | flex-2,`font-mono-c text-sm text-muted-fg` | URL,部分桶可空(显示 `—`) |
| 状态 | 计算字段 | `buckets.list.column.status` | `w-28` | Tag:`已添加 --success` / `未添加 --info`(§2.4 业务状态色) |
| 操作 | — | — | `w-24` 右对齐 | `[Btn 添加 default sm]` **或** `[Tag 已添加]`(互斥) |

`状态` 列计算逻辑:遍历 `scoop:buckets:list` 的 `name` 集合,命中 → `已添加`(disabled 不可再添加);未命中 → `未添加`(可添加)。这与 PRD §5 验收点"已知区段中已添加的桶不应再作为可添加对象展示"严格对应。

操作 cell 三态:
- 已添加 → `<Tag variant="outline">已添加</Tag>`(无按钮,点击无反应)
- 未添加 + 非 running → `<Btn 添加 default size=sm>添加</Btn>`(lucide `Plus`)
- 未添加 + 有任务 running(P08 开启) → 整行 `opacity-50 cursor-not-allowed`,按钮 disabled(§8.4)

### 3.5 AlertDialog · 移除确认(场景 4)

不可逆操作必须二次确认(§8.2 硬约束 + PRD §1 可恢复性原则)。

```
┌── AlertDialog (max-w-md, 居中) ─────────────────────────────┐
│ 移除桶 · Remove bucket                                     │
│ ───────────────────────────────────────────────────────── │
│ [Trash2 图标] 确认移除桶 `extras`?                         │
│                                                             │
│ 移除后,extras 桶内所有可装软件包将不再作为可装范围展示。    │
│ 已通过 extras 安装到本机的应用不受影响。                    │
│                                                             │
│                              [取消 outline]  [移除 destructive]│
└─────────────────────────────────────────────────────────────┘
```

- 主按钮文案:`移除` / `Remove`,`variant=destructive`(§7.1)
- 次按钮:`取消` / `Cancel`,`variant=outline`
- Esc = 取消,Enter = 主操作(§8.8)
- 点击"移除" → 触发 `scoop:buckets:remove({ name })`,打开 P08(场景 4 后续 / §8.12)

```
┌── AlertDialog (max-w-md) ────────────────────────────────────┐
│ 添加桶 · Add bucket                                          │
│ ────────────────────────────────────────────────────────────│
│ [Plus 图标] 确认添加桶 `versions`?                           │
```

- 仓库 URL 默认值来自 `KnownBucket.repo`,若为空则提示"使用该桶官方默认仓库"
- 点击"添加" → 触发 `scoop:buckets:add({ name, repo })`,打开 P08

### 3.7 P10 空状态(已添加 Tab)

PRD §5:已添加桶列表为空时,应显示 P10 空状态并引导用户前往已知区段添加桶。

```
┌── 空状态 Card ──────────────────────────────────────────────┐
│  ┌────────┐                                                  │
│  │ 桶图标 │  暂无已添加桶                                     │
│  │        │  本机尚未添加任何桶,请前往"已知桶"添加默认桶。      │
│  └────────┘  [Btn 前往已知桶 →  default lg]                  │
└──────────────────────────────────────────────────────────────┘
```

- 居中布局(`flex flex-col items-center justify-center`)
- 图标:`lucide PackageOpen` 64×64,`text-muted-fg`
- 主操作按钮:点击切换到"已知"Tab(行为由前端状态机处理,**不**触发 IPC)

### 3.8 P10 错误状态(已知 Tab · PRD §5)

已知桶清单读取失败时(`E_SCOOP_SPAWN_FAILED` / `E_SCOOP_PARSE_FAILED`):

```
┌── 错误状态 Card ─────────────────────────────────────────────┐
│  ┌────────┐                                                  │
│  │ 警示框 │  已知桶清单读取失败                                │
│  │ 红色   │  [错误码] E_SCOOP_SPAWN_FAILED · 中文翻译          │
│  └────────┘  [Btn 重试 →  default]    [Btn 查看日志 →  outline]│
└──────────────────────────────────────────────────────────────┘
```

- 严重错误用 Dialog(§8.5);本场景使用页面内嵌 Card(因需保留 Tab 切换能力,允许用户回退到已添加 Tab)
- 重试按钮:点击重新调用 `scoop:buckets:known`(不重读已添加 Tab)

### 3.9 P08 进度对话框(场景 3 / 4 · §9.4)

非 modal、可关闭、长时操作期间本页其他按钮禁用(§8.4)。

| Tab | 操作 | stage(API §2.4) |
| --- | --- | --- |
| 已知 → 添加 | `cloning` | "克隆中" / "Cloning" |
| 已添加 → 移除 | `removing` | "移除中" / "Removing" |

P08 完整三段式(进度条 + 阶段文本 + 原始日志)由 P08 原型负责,本页 **不** 重复实现;此提示词仅约束"打开 P08 时,本页 Tabs 行内按钮全 disabled、刷新按钮 disabled"。

### 3.10 失败重试(场景 5 · PRD §4)

P08 返回 `E_BUCKET_ADD_FAILED` / `E_BUCKET_REMOVE_FAILED` / `E_SCOOP_INSTALL_TIMEOUT` / `E_JOB_CANCELLED`:

- **Cancelled**:不展示为失败;目标桶回到动作前的业务状态,Toast success 文案"已取消添加/移除"(i18n key:`buckets.manage.toast.cancelled`)
- **Failed**:页面顶部出现一条 inline 错误条(`bg-destructive-soft border-l-4 border-destructive-c`,**只此一处使用 inline 红条**,因 PRD §5 验收要求"应看到失败原因与重试入口",其他场景严格遵守 §8.5 禁 inline alert)
  - 内容:错误码徽标 + i18n 翻译文案 + `[重试 default sm]`
  - 重试按钮:点击重新调用上次失败的 IPC
  - 状态保持:失败桶业务状态不动(已添加列表中的桶移除失败 → 仍在已添加列表,仅显示错误条)

## 5. 状态切换矩阵(覆盖 PRD §4 全部场景)

| 场景 | PRD §4 | 默认 Tab | 表格内容 | 覆盖层 | 备注 |
| --- | --- | --- | --- | --- | --- |
| 1. 查看已添加 | §4 场景 1 | 已添加 | N 行(已添加桶) | — | 默认进入态 |
| 2. 查看已知 | §4 场景 2 | 已知 | N 行(已知桶,标记状态) | — | — |
| 3. 添加桶(确认) | §4 场景 3 | 已知 | 高亮目标行(可加) | AlertDialog 添加确认 | 点击"添加"打开 |
| 3'. 添加桶(进行) | PRD §3.2 | 已知 | 目标行 `opacity-50 disabled` | P08 进度对话框 | §8.4 其他按钮禁用 |
| 4. 移除桶(确认) | §4 场景 4 | 已添加 | 高亮目标行 | AlertDialog 移除确认 | 点击"移除"打开 |
| 4'. 移除桶(进行) | PRD §3.2 | 已添加 | 目标行 disabled | P08 进度对话框 | — |
| 5. 添加/移除失败 | §4 场景 5 | 失败发生时的 Tab | 状态保持(动作前的业务状态) | 错误条 inline | 错误条含重试 |
| 6. 已添加为空 | PRD §5 验收 | 已添加 | P10 空状态 | — | 主按钮"前往已知桶" |
| 7. 已知读取失败 | PRD §5 验收 | 已知 | P10 错误状态 | — | 重试 + 查看日志 |

> **PRD §4 场景 5** 强调:"添加或移除失败时,本页保留失败对象的业务状态"。意味着错误条是页面顶部 inline 元素,不替换表格内容,目标桶业务状态完全保持(已添加列表中失败的桶不被移除)。

## 7. 交互细节

### 7.1 Tab 切换(§3.2 Tabs)

- 切换 Tab 时:**仅** 重新读取目标 Tab 的数据(若 `lastFetched` 超过 30s,否则复用缓存)
- 切换瞬间:目标 Tab 区域显示骨架占位(`bg-muted animate-pulse`,与 P04/P05 一致),避免空白闪烁
- 两 Tab 的"已添加"判定共用同一份已添加桶缓存,避免数据漂移

### 7.2 行点击 vs 按钮点击(§8.3)

- 行 `<tr>` `cursor-pointer`,`onClick` 打开对应 AlertDialog
- 操作 cell 内 `<button>` `e.stopPropagation()`,仅触发按钮对应动作
- 已添加 Tab:`移除` 按钮 = destructive,点击直接开移除确认 AlertDialog
- 已知 Tab:`添加` 按钮 = default;`已添加` Tag 无 click handler(纯展示)

### 7.3 长时操作期间禁用(§8.4)

P08 打开时:
- Tabs 触发器 `disabled`
- 刷新按钮 `disabled`
- 所有行内 `添加` / `移除` 按钮 `disabled`(`opacity-50 cursor-not-allowed`)
- 已运行的目标行 `bg-info-soft` 高亮(可视化"正在操作此桶")
- AlertDialog 不可再打开(已开则正常)
- 切换 Tab 不允许(tabs disabled)

P08 关闭后:目标 Tab 自动重读(API 文档 §3 已添加/已知两端联动),无需用户手动刷新。

### 7.4 取消态语义(§3.9 PRD §5 验收点)

P08 返回 `state=cancelled`:**不** 走失败错误条,**不** 标记目标桶为失败状态。展示右下 Sonner Toast(`success` variant,绿色):
- `已添加 → 已取消添加`
- `已移除 → 已取消移除`

### 7.5 失败状态保持(§3.10 / PRD §5 验收点)

P08 返回 `state=failed`:
- 顶部 inline 错误条(`bg-destructive-soft border-l-4 border-destructive-c`,§8.5 例外允许)
- 错误条 5s 内不自动消失,直到用户主动关闭(×)或重试成功
- 目标桶业务状态完全保持(已添加 Tab 中移除失败 → 桶仍在表中;已知 Tab 中添加失败 → 桶仍是"未添加"状态)
- 错误条"重试"按钮:重新调用上一次失败的 IPC,**不** 重置表格其他数据

### 7.6 不可逆操作二次确认(§8.2)

仅"移除"需要二次确认(添加非破坏性,可由用户重新添加回来)。移除使用 AlertDialog,文案明确写出目标桶名(`{name}` 占位符替换),并说明业务影响(可装范围收缩),与设计原则 §1 "可恢复性"一致。

### 7.7 键盘交互(§8.8)

- `Esc` → 关闭当前打开的 AlertDialog(取消)
- `Enter` → AlertDialog 主操作(确认添加 / 确认移除)
- Tab 内部 Tab 键 → 表格行 → 操作按钮(原生顺序)

### 7.8 空 vs 错误边界(PRD §5)

- 已添加 Tab 数据为空数组 → P10 空状态,**不是** 错误
- 已知 Tab 数据为空数组 → **不** 视作空状态(可能官方未公布),显示"暂无已知桶"提示(灰色,非 error)
- `scoop:buckets:list` 调用失败 → 错误条 + 重试(不阻塞切换到已知 Tab)
- `scoop:buckets:known` 调用失败 → P10 错误状态(占满整个 Tab 内容区)
