# P10 · 空状态与错误页 · 设计提示词

> 文档层级:页面级设计提示词(designer 产出,登记不送审)
> 适用范围:scoop-gui 项目 electron 端 P10(空状态与错误页)对应的 HTML 原型与生产实现
> 上游依赖:
> - `docs/design/systems/electron.md`(v2,端级 token / §8 硬约束 / §9 网格+辉光+渐变线)
> - `docs/prd/pages/electron/scoop-gui/P10-空状态与错误页.md`(已审批,§4 七大场景 / §5 验收)
> - `docs/architecture/api/electron/scoop-gui.md`(已审批,15 个 IPC 通道与错误码)

## 1. 设计目标(v2)

P10 不是独立顶级页面,而是 P03/P04/P05/P06 的**子状态层**:当数据为空、读取失败、Scoop 整体不可用、长时任务失败时,统一接管当前页面内容区。设计目标:

1. **不出现空白页面**:任何"读不到 / 没有"的分支都必须呈现友好说明 + 主操作,让用户始终知道"下一步能做什么"。
2. **业务语言,不暴露堆栈**:所有错误文案使用业务术语描述影响(如"无法读取已装软件包"),错误码以小字形式附在主文案下方,不展示原始堆栈。
3. **可恢复性优先**:每个错误态至少给一个明确主操作(重试 / 返回 / 清空搜索),主 CTA **必带 `var(--glow-primary)` 微光**(v2 科技感硬约束,见 `electron.md §9`)。
4. **i18n 一致**:中英两套文案通过 `t()` 调用,**禁止半中半英**,与 P09 语言切换立即同步。
5. **不引入范围外功能**:P10 的引导动作仅限 PRD §4 列出的"重试 / 清空搜索 / 返回主界面 / 前往桶管理 / 查看帮助",不引导登录、配置编辑、备份还原、系统通知等阶段外能力。

## 2. 子状态类型清单(对应 PRD §4)

> 七大场景分别映射到一种"子状态类型";每种类型对应**一种视觉模板**,差异在文案、图标、按钮与辉光色。

| 类型 ID | PRD §4 场景 | 来源页面 | 严重度 / tone | 触发条件 |
| --- | --- | --- | --- | --- |
| `EMPTY_INSTALLED` | 场景 1 | P04 | 空 / info | `scoop:apps:listInstalled` 返回空数组 |
| `EMPTY_AVAILABLE` | 场景 2 | P05 | 空 / info | `scoop:apps:search('')` 返回空数组 |
| `EMPTY_SEARCH` | 场景 3 | P05 | 空 / info | `scoop:apps:search(query)` 返回空数组且 query 非空 |
| `EMPTY_BUCKETS` | 场景 4 | P06 | 空 / info | `scoop:buckets:list` 返回空数组 |
| `READ_FAILED` | 场景 5 | P03/P04/P05/P06 | 错误(可重试) / warn | 任一读通道返回 `{ ok: false }`,错误码属 `E_SCOOP_SPAWN_FAILED` / `E_SCOOP_PARSE_FAILED` |
| `SCOOP_UNAVAILABLE` | 场景 6 | P03 | 错误(需引导) / destructive | `scoop:onboarding:check` 在主区运行时返回 `available: false` 或读通道返回 `E_SCOOP_NOT_FOUND` |
| `JOB_FAILED` | 场景 7 | P08 结束后回到来源页 | 错误(可重试) / destructive | `ProgressEvent.state === 'failed'` 且 `error.code` 属 `E_SCOOP_INSTALL_FAILED` 等写操作失败码 |

> 七种类型**穷尽 PRD §4 全部场景**,原型必须全部覆盖。
> tone → 视觉:
> - `info` = 主色 `--primary` 微光 + `--fg-muted` 描述
> - `warn` = `--warning` 图标 + 错误码
> - `destructive` = `--destructive` 图标 + `--destructive-glow` + 错误码

### 4.4 文案禁忌(原型自检)

- ❌ "Oops" / "哎呀" / "抱歉出错啦" — 与桌面工具感冲突,见 `electron.md §1` 简洁原则
- ❌ 直接展示 `cause` 堆栈 / scoop CLI 原始 stderr(仅主进程日志记录,见 `electron.md §8.5`)
- ❌ 把"用户取消"描述为失败:`E_JOB_CANCELLED` 走 P08 终态呈现,**不在 P10 落点**(见 PRD §5 验收第 11 条)
- ❌ 中英混杂:切换语言后整页必须单一语言

## 5. 操作矩阵(主操作 + 次操作)

> 主操作 = 用户最可能想做的事,`variant=default`(`--primary` 实心 + `var(--glow-primary)`);次操作 = 次要入口,`variant=outline`(`--bg-overlay` + `--border` 边框)。
> 不允许"主操作 + 另一个主操作";最多一主一次,保证视觉层级清晰。

| 类型 ID | 主操作 | 次操作 | 错误码展示 |
| --- | --- | --- | --- |
| `EMPTY_INSTALLED` | "前往浏览"(→ P05) | — | 否 |
| `EMPTY_AVAILABLE` | "前往桶管理"(→ P06) | "重试读取" | 否 |
| `EMPTY_SEARCH` | "清空查询"(→ P05 重置) | — | 否 |
| `EMPTY_BUCKETS` | "查看已知桶"(→ P06 切换 tab) | — | 否 |
| `READ_FAILED` | "重试" | "返回主界面"(→ P03) | 是 |
| `SCOOP_UNAVAILABLE` | "重试读取" | "返回主界面"(→ P03) | 是 |
| `JOB_FAILED` | "重试" | "放弃"(→ 来源页) | 是 |

> "重试"动作:读通道失败时,重新触发对应 IPC 调用;写通道失败时,回到来源页重新进入 P08(见 PRD §3.3)。

## 7. 布局约束(沿用 `electron.md §8` v2)

1. **不独立导航**:P10 不在 Sidebar 出现;仅作为 P03/P04/P05/P06 的内容区替换(见 PRD §5 验收第 1 条)
2. **窗口尺寸**:默认 `1280×800`,最小 `1024×600`(沿用 §8.1);卡片随窗口缩放居中
3. **侧边栏**:空状态/错误态时侧边栏保持可点击,**允许切换到其他可用页面**(如 `SCOOP_UNAVAILABLE` 时侧边栏仍可用,见 PRD §5 验收第 7 条)
4. **顶部品牌线**:全应用顶部 1px `linear-gradient(to right, transparent, var(--primary), transparent)`,贯穿整个 1280px 宽度
5. **网格 + 辉光**:PageHeader 以下 / Content 区上半部分叠加 `.bg-grid` + `.bg-glow-top`,营造终端科技感
6. **主题切换**:P09 主题切换按钮始终可点;v2 默认主题为暗色(`--bg` `#0B1220`),亮色作为备选(由 P09 §3 控制,不在本原型单独演示)
7. **键盘**:空态/错误态页面无焦点陷阱;主按钮获取默认焦点(`autofocus`),Enter 触发主操作(沿用 §8.8)
8. **滚动**:页面内容区不溢出;卡片在窄窗口(< 600px)允许垂直滚动(满足最小尺寸即可)

## 8. 错误码映射

> 错误码由 IPC 返回值 `error.code` 决定(见 `electron.md §8.5` + `api/electron/scoop-gui.md §3`);P10 只展示给用户,**不暴露 cause**。

| 类型 ID | 可能出现的错误码 |
| --- | --- |
| `READ_FAILED` | `E_SCOOP_SPAWN_FAILED` / `E_SCOOP_PARSE_FAILED` |
| `SCOOP_UNAVAILABLE` | `E_SCOOP_NOT_FOUND` / `E_SCOOP_VERSION_PARSE_FAILED` / `E_SCOOP_SPAWN_FAILED` |
| `JOB_FAILED` | `E_SCOOP_INSTALL_FAILED` / `E_SCOOP_PERMISSION_DENIED` / `E_SCOOP_INSTALL_TIMEOUT` / `E_BUCKET_ADD_FAILED` / `E_BUCKET_REMOVE_FAILED` |

> `E_JOB_CANCELLED` 不进入 P10(走 P08 终态"已取消"分支,见 PRD §5 验收第 11 条 + `electron.md §8.4`)。
