## 1. 页面定位

P01 是**首启独立页**,**不是** P03 主界面壳的某个路由。PRD §3.1 / §3.3 明确:启动序列完成后,若 `scoop:onboarding:check` 返回 `available: false`,停留在此页;若 `available: true`,直接进入 P03 而**不显示**本页。

因此:
- **不渲染 Sidebar / PageHeader**(P03 主界面壳骨架的组件,见设计系统 §9.1)
- **占满整个窗口**(1280×800,见设计系统 §9 硬约束)
- 单页分上下两块:**检测态块**(轻量)+ **安装配置块**(常态)+ **失败重试块**(错误态)
- **底层窗口背景**叠加 `.bg-grid` + `.bg-glow-top`(v2 §2.1 科技感)+ 顶部 1px `from-transparent via-primary to-transparent` 渐变线(v2 §9)

## 4. 关键组件(v2 形态)

### 4.1 顶部 Title bar

- 高度 44px,`background: rgba(11,18,32,0.7) + backdrop-filter: blur(12px)` 玻璃感
- 左侧:run-green 渐变 Logo(22×22,`box-shadow: var(--glow-primary)`)+ `scoop-gui`(Space Grotesk 600 13px)+ `[P01]` tag
- 右侧:语言切换(zh/EN `lang-toggle`)+ 窗口控件(─/□/×,hover destructive 仅 ×)
- 整条 `app-region: drag`(electron 拖动区);窗口控件 `no-drag`

### 4.2 检测状态 Card

基于 v2 §8.2 Card:背景 `--bg-elevated`,圆角 `--radius-lg` 12px,边框 1px `--border`,阴影 `--shadow-md`,**顶部 thin gradient line** `from-transparent via-primary/40 to-transparent`(v2 §8.2 科技感点缀)。

| 场景 | 视觉 | 语义色 |
|---|---|---|
| 检测中(场景 1) | Spinner(`Loader2` 1.5px stroke + `box-shadow: var(--glow-primary)`)+ checklist(执行步骤 4 行,micro-anim 脉冲当前项)+ mono log hint | `--primary` |
| 未检测到(场景 2) | `AlertTriangle` icon + 原因说明 + 右上"重新检测" secondary 按钮 + mono 错误摘要框(0.06 红透 + 1px 红边) | `--warning` |
| 检测失败(场景 2 子态) | `XCircle` icon + "检测过程异常" + 重试入口 | `--destructive` |
| 安装失败(场景 5 失败重试) | `XCircle` icon + `tag err` 错误码 + 副标题含复检未通过说明 | `--destructive` |

**checklist 微观规范**(state 1 专用):
- 已完成项:实心 `dot` + `--primary`,文本 `--fg`
- 进行中项:实心 `dot` + `animation: pulse 1.2s`,文本 `--primary`
- 待办项:空心 `dot` `--border-strong`,文本 `--fg-muted`
- 右侧 `cmd` 部分(命令 / 路径)用 mono 12px `--fg-subtle`,`margin-left: auto`

### 4.3 安装配置 Card

8 项配置(PRD §2 功能 2 强制要求,§5 验收点禁止增减):

| 字段(API §2.3) | UI 类型 | 默认值(对齐 `install.ps1`) |
|---|---|---|
| `ScoopDir` | Input + `~/` 前缀(mono) | `scoop` |
| `ScoopGlobalDir` | Input + `~/` 前缀(mono) | 留空(用默认) |
| `ScoopCacheDir` | Input + `~/` 前缀(mono) | `scoop/cache` |
| `NoProxy` | Switch(`--radius-full`,h=20) | `false` |
| `Proxy` | Input(mono) | 空 |
| `ProxyCredential.username` | Input | 空 |
| `ProxyCredential.password` | Input type=password | 空 |
| `ProxyUseDefaultCredentials` | Switch | `false` |
| `RunAsAdmin` | Switch | `false` |

布局策略(v2 调整):
- 字段分组:3 个区段 → "路径配置 · Paths" / "网络代理 · Proxy" / "权限 · Privilege",区段标题 `text-xs uppercase letter-spacing:0.05em` + 左侧 3px `--primary` 短杠带 `--glow-primary`
- `grid grid-cols-2 gap-3`(12px gap,§4 间距 token `3`)
- 每个 Input:
  - 高度 40px(§8.3)
  - 背景 `--bg-overlay`
  - 边框 1px `--border`
  - focus:`border-color: var(--primary)` + `box-shadow: var(--ring-primary)`(1px 主边 + 4px 主色微光外环)
  - hover:`border-color: var(--border-strong)`
  - invalid:`border-color: var(--destructive)` + `box-shadow: 0 0 0 1px var(--destructive), 0 0 0 4px var(--destructive-glow)`
- 路径类字段前缀 `~/`(mono 11px `--fg-subtle`),Input 内 `padding-left: 38px`,路径值用 mono 12px
- 每个 Label 右侧 `[label-key]` mono 11px tag(`bg-overlay` + 1px `--border` + `--fg-subtle`),提示 i18n key 命名严格对齐 PRD §5 / API §2.3
- hint 文案:`text-xs text-fg-muted`,内嵌 `code` mono 11px

### 4.4 全局目录权限提示(场景 4)

当 `ScoopGlobalDir` 非空 **或** `RunAsAdmin` 启用时,自动展开 inline alert(非 Toast,非 Dialog):

```
border-left: 3px solid var(--warning)
background: rgba(245,158,11,0.08)
padding: 12px 14px, radius: --radius-md
icon: ShieldAlert (--warning)
title: 该配置可能需要管理员权限
desc: 已启用 RunAsAdmin 并填写 ScoopGlobalDir, 将触发 UAC 提权…
```

样式 = v2 风格"半透明 + 警告色左边线",**不是** destructive red alert(语义不冲突,见 PRD §4 场景 4 + 设计系统 §8.5 "禁用 inline alert 红条",此条为黄色警告条例外)。

**RunAsAdmin toggle-row**(场景 4 启用态):边框变 `rgba(245,158,11,0.4)`,背景 `rgba(245,158,11,0.04)`,label 文本 `--warning`,以视觉强化"已开启需提权"。

### 4.5 Footer Action Bar

右对齐,固定结构:

```
[meta-left: jobId / 状态 dot + mono 11px]                      [Btn 取消 outline] [Btn 主操作]
```

**长时操作期间**(进入 P08,state 5):本按钮 `disabled`(`opacity:0.5 cursor:not-allowed filter:grayscale(0.4)`,§9 硬约束),且**整页 `.card` `.input` `.toggle-row` 全部 dim**(原型用 `opacity:0.55 pointer-events:none` 模拟)。

| 状态 | 主 CTA 文案 | 主 CTA 视觉 |
|---|---|---|
| 1 检测中 | (隐藏) | — |
| 2 未检测到初次 | "协助安装" | primary lg + 微光 + → icon |
| 3 填写配置 | "确认安装" | primary lg + 微光 + ✓ icon |
| 4 全局目录 | "确认安装(管理员)" | primary lg + 微光 + shield icon |
| 5 安装中 | "确认安装" disabled | primary lg 灰显 |
| 6 失败重试 | "重新安装" + "修改配置" secondary | primary lg + 微光 + ↻ icon |

主 CTA 必带 `box-shadow: var(--glow-primary)`(v2 §8.1 主色实心按钮硬要求)。hover 增强到 `--primary-hover` + brightness 提升。

### 4.6 失败重试 Card(场景 5)

仅在 `scoop:onboarding:install` 返回失败(`E_ONBOARDING_INSTALL_FAILED` / `E_ONBOARDING_POSTCHECK_FAILED` / `E_SCOOP_INSTALL_TIMEOUT` / `E_SCOOP_PERMISSION_DENIED`)时显示:

- 头部:`XCircle` icon(`--destructive`)+ "安装失败 · 未检测到 Scoop" + 副标题说明复检情况 + 右上 secondary "重新检测" 按钮
- **失败摘要 Card**(新结构,v2 强化):`pre` 元素,mono 12px,`bg: rgba(239,68,68,0.06)` + 1px `rgba(239,68,68,0.20)` 边框,内容 5 行结构化:
  - `[code]` 错误码(`--destructive`)
  - `[stage]` 失败阶段(`--fg-muted`)
  - `[time]` 时间戳 + elapsed(`--fg-muted`)
  - `[msg]` 终端输出摘要(`--fg-muted`)
  - `[hint]` 业务建议(`--fg-muted`)
- 不展示 `cause` 字段原始堆栈(§9 硬约束:仅日志,UI 不暴露)
- 同时伴随右上 Toast(`toast.err`,`border-left:2px destructive`,`box-shadow: var(--shadow-lg) + var(--glow-destructive)`,6s 自动消失,见设计系统 §8.10)

### 4.7 P08 长时操作进度对话框(state 5 · 非 modal)

> P08 是 P01 的**子对话框**,不是 P01 自身的组件。本提示词仅展示 P01 内部调用 P08 的视觉编排;P08 自身详见 `P08-长时操作进度对话框.md`。

P01 进入 installing 状态时:
- P01 主体所有可交互元素 `opacity: 0.55 pointer-events: none`
- P08 以 `position:absolute; inset:0` 浮层遮罩(`bg-black/35 + backdrop-blur 6px`),**非 modal**(仍能看到 P01 背景)
- P08 卡片:640px 宽,`bg-elevated`,`radius-xl`(16px),`shadow-xl`
- 顶部:`thin-gradient-bar`(running 时)2px `from-primary via-accent to-primary` + `box-shadow: var(--glow-primary)`
- 内容:title `scoop 协助安装` + `[RUNNING]` badge(`bg-primary/12` + `1px primary/30`,mono 10px uppercase)+ 当前阶段 mono 12px(`--primary`)+ 关闭 × 始终可见
- 进度条:`h-2` 圆角胶囊,容器 `--bg-overlay` + 1px `--border`,填充 `linear-gradient(primary → primary-hover)` + `box-shadow: var(--glow-primary)`(v2 §8.5 硬要求)
- 日志区:`h-48 overflow-auto bg-bg/50 font-mono text-xs`,stage 行 `--accent`,ok 行 `--primary`,warn 行 `--warning`,err 行 `--destructive`,行首 mono 时间戳 `--fg-subtle`
- Footer:`meta mono 11px`(jobId / channel / elapsed)+ `[关闭(继续后台) ghost]` + `[取消 outline · --fg-muted]`
  - **取消按钮禁用 destructive 色**(v2 §9 "取消 ≠ 失败"),用 `--fg-muted` 中性灰

### 4.8 错误 Toast(state 6 失败重试)

`border-left: 2px var(--destructive)` + `box-shadow: var(--shadow-lg) + var(--glow-destructive)`,位置 `top: 60px right: 24px`(标题栏下方),6s 自动消失。失败日志同步落到结构化日志(developer 实现),UI 仅展示业务语言。

## 7. 交互细节

### 7.1 表单校验(对齐 PRD §5 验收点)

- **路径无效**(空 / 含非法字符):Input `border-color: var(--destructive)` + `box-shadow: 0 0 0 1px destructive + 0 0 0 4px destructive-glow` + 下方 `field-error`(mono-ish,12px destructive,带 ✕ icon)
- **代理冲突**:`NoProxy = true` 且 `Proxy` 非空 → 下方文案 "已启用'不使用代理',无需填写代理地址。"
- **代理凭据不完整**:`Proxy` 非空 且 `ProxyCredential` 用户名为空 → 下方文案 "已填写代理地址,请补全用户名与密码。"

校验在 footer "确认安装" 按钮点击前完成,失败则按钮 disabled 并显示聚合错误条(可复用 state 6 的失败重试 Card 结构)。

### 7.2 不可编辑态(安装中 · state 5)

P08 打开时,本页面所有 Input / Toggle / Button **disabled**(`opacity: 0.55 pointer-events: none filter: grayscale(0.4)`)。
Card 头部追加 `tag dim "已锁定"`(mono 11px)。用户在 P08 关闭前不可修改本次已确认的安装配置(PRD §5 验收点)。

### 7.3 配置复用(PRD §5 验收点)

`Preferences.scoopInstallConfig`(API §2.3)在确认安装后写入,下次进入 P01 自动填充 + 顶部 meta 显示"上次配置: 2026-07-10 14:23 (已写入偏好)"。
state 6(失败重试)在配置 Card 标题旁加 `tag dim "已自动填充"`,副标题说明"可直接重试或修改后重试"。
此项由 developer 在实现期接入 `scoop:prefs:get`,原型仅占位。

### 7.4 v2 视觉一致性

- 标题统一用 Space Grotesk(heading),不用系统字体
- 所有路径 / 错误码 / jobId / 时间戳 / 命令统一用 JetBrains Mono
- 主 CTA / 选中状态 / 进度条填充 / 运行点 必带 `--glow-primary`(科技感硬要求)
