## 1. 页面定位

P05 是 **P03 主界面壳的子路由**(`/browse` 或 `/apps`,由 developer 决定),不是独立页。入口与日常浏览流程见 `flows/scoop-gui.md` §1.2,业务结果刷新见 §1.3。

因此:
- **必须渲染 P03 主界面壳**:Sidebar(240px,固定)+ Content(flex-1),见设计系统 §9.1
- **页面区占主区 1040px**(1280 - 240)
- **首屏即功能**,无 hero / 无 onboarding(已在 P02 完成)

页面职责:**桶内软件包浏览与搜索**。展示已添加桶范围内的软件包(可装未装 + 已装),通过搜索定位目标,引导进入 P07 详情或发起 P08 安装。

## 3. 关键组件(引用设计系统 §7)

### 3.1 Sidebar(240px,固定)

沿用 P03 主壳(§9.1):

- 折叠态宽度 48px,展开态 240px(§8.1)
- 导航项高度 `h-10`,激活态使用 `--secondary` 背景
- 当前路由 P05 高亮显示
- 底部 ThemeToggle(三选项:亮/暗/跟随系统,§8.7),P09 也提供切换入口

### 3.2 SearchBar(cmdk 风格)

顶部 PageHeader 内,横跨主区宽度。**视觉参考 cmdk**(§7.5),但**常驻而非弹出**(全局搜索由 Cmd+K 触发,§8.8):

- 高度 `h-10`(与其他输入框一致,§7.2)
- 圆角 `rounded-md`(8px,§5)
- 背景 `--background`,边框 1px `--border`,focus `ring-2 ring-ring ring-offset-2`(§7.2)
- 左侧:`Search` 图标(lucide `Search`,1.5px stroke,§8.10)
- 中央:占位文案随状态变化:
  - 空查询:"搜索已添加桶中的软件包"(zh) / "Search apps in added buckets"(en)
  - 有查询:显示当前关键字
- 右侧:⌘K / Ctrl+K 快捷键徽标(`text-xs text-muted-foreground`,内嵌 `<kbd>` 样式),提示用户可用快捷键聚焦/打开全局面板
- 清空按钮:有查询时右侧出现 `X` 图标(§8.8 Esc 也清空,但保留可见入口)

> 备注:`scoop:apps:search` 接受空 query 等价于"列出全部已添加桶内可装应用"(API §4.2),故清空查询即为"全部列表"。

### 3.3 Toolbar(h-12)

| 元素 | 位置 | 视觉 | 备注 |
| --- | --- | --- | --- |
| 状态 Tabs | 左 | `TabsList h-10`(§7.9) | 触发 Tabs:全部 / 可装未装 / 已装(本地维护,无需 IPC) |
| 桶范围徽标 | 左 Tabs 后 | Tag `rounded-sm text-xs`(§5) | 例:"4 个桶 · 共 38,402 个包";数字按桶加载实时更新 |
| 排序 | 右 | DropdownMenu(`btn-outline sm`) | "按名称 / 按更新时间";本地状态 |
| 刷新 | 极右 | IconButton(`btn-ghost icon`,lucide `RefreshCw`) | 触发 `scoop:apps:search('')` 重新拉取 |

### 3.4 Table(主体,§7.8)

| 列 | 宽度策略 | 内容 | 视觉 |
| --- | --- | --- | --- |
| Name | 弹性 | `AppInfo.name`(§2.2 API),行内可含可选状态 Tag(若已装) | `text-sm font-medium`(§3.2) |
| Version | 100px | `AppInfo.version`,可装未装时显示 `—`(em dash,§3.4) | `text-sm font-mono-c`,`text-muted-fg` |
| Source | 96px | `AppInfo.source`(桶名,如 `main` / `extras`) | Tag `rounded-sm text-xs`(§5) |
| Updated | 140px | `AppInfo.updated`,缺失时 `—` | `text-sm text-muted-fg` |
| Info | 弹性 | `AppInfo.info`,可选,过长截断 | `text-sm text-muted-fg truncate` |
| 操作 | 120px(固定) | 按钮列 | 见 §3.5 |

#### 3.4.1 行交互(§8.3)

- **整行 `h-10`,可点击**:cursor-pointer,点击空白区域 → 触发详情(进入 P07,§3.5)
- **hover**:`bg-accent`(§7.8)
- **列分隔**:仅水平线,无垂直线(§7.8)
- **行内按钮**:点击按钮区不触发行点击,事件冒泡由 developer 控制(`stopPropagation`),互不串扰

### 3.5 操作按钮(关键,§8.4)

操作按钮列宽 120px,根据软件包状态切换:

| 状态 | 按钮变体 | 文案(zh / en) | 备注 |
| --- | --- | --- | --- |
| 可装未装(Available) | `default`(`h-8 px-3`,§7.1) | 安装 / Install | 点击 → `scoop:apps:install` |
| 安装中(Running) | `outline`(`h-8 px-3`)+ Spinner(lucide `Loader2` animate-spin) | 安装中… / Installing… | `disabled`,P08 进行中 |
| 已装(Inactive) | `secondary`(`h-8 px-3`) | 已装 / Installed | `disabled opacity-50 cursor-not-allowed`(§8.4) |
| 失败(Failed) | `destructive`(`h-8 px-3`) | 重试 / Retry | 上一轮 P08 终态失败,点击重发 |

按钮宽度统一 88px(`w-[88px]`),保证列对齐;Spinner 在 "安装中" 状态文字左侧 12px(`gap-2`)。

**多次提交防护**(§8.4):同一按钮在请求未 resolve 时禁用,避免重复触发 `scoop:apps:install`。

### 3.6 P10 子状态(空结果 / 错误)

无结果(搜索无匹配)、桶内无可装(场景 6)、读取失败时,Table 区替换为 P10 子状态:

| 子态 | 触发 | 视觉 |
| --- | --- | --- |
| 无匹配结果 | `search(query) === []` 且 query 非空 | `SearchX` 图标(lucide)+ 文案 + `[Btn 清空查询 outline]` |
| 桶内为空 | `search('') === []`(已添加桶但无可装包) | `PackageOpen` 图标 + 文案 + `[Btn 管理桶 secondary]` → 跳 P06 |
| 读取失败 | IPC 错误(`E_SCOOP_SPAWN_FAILED` / `E_SCOOP_PARSE_FAILED`) | `AlertTriangle` 图标 + 错误码 + `[Btn 重试 default]`(触发重新拉取) |

样式:垂直水平居中,占满 Table 区,图标 48px,文案 `text-sm text-muted-fg`,按钮组 `mt-4`(§4)。**不用 inline alert 红条**(§8.5 硬约束)。

### 3.7 全局快捷键(§8.8)

- `Cmd/Ctrl + K`:聚焦 SearchBar;MVP 阶段不展开全局 cmdk 面板(预留入口,§8.8)
- `Esc`:清空 SearchBar 查询(聚焦时)
- `↑/↓`:Table 内行间键盘导航(沿用 Radix Table 默认,§8.9)
- `Enter`:行高亮时 → 进入 P07

## 7. 交互细节

### 7.1 搜索行为(对齐 PRD §5 验收点)

- **空 query = 全部列表**:进入页面时调用 `scoop:apps:search({ query: '' })`(API §4.2:空字符串等价于"列出全部已添加桶内可装应用")
- **输入 query → 实时触发**:300ms debounce 后调用 `scoop:apps:search({ query })`(避免每键触发 IPC,developer 优化)
- **清空 query → 回到全部**:点击 SearchBar 右侧 `X` 或按 Esc → query = '' → 重新触发 search('') 或恢复上次结果(developer 决定缓存策略)
- **搜索范围**:严格限于"已添加桶",`scoop search` 原生支持(API §4.2)

### 7.2 状态 Tab 与搜索的优先级

- **搜索时(query 非空)**:状态 Tab 高亮"全部"(Tab 只对"无 query 时"生效,作为另一维度的过滤);搜索优先级高于 Tab
- **无搜索时(query === '')**:Tab 生效,过滤"全部 / 可装未装 / 已装"
- 这样避免 UI 上两个过滤器冲突,符合桌面工具的"少选项"(§1)

### 7.3 行点击 vs 按钮点击(§8.3)

- 行空白区域(cursor-pointer 整行):进入 P07
- 行内"安装"按钮:`stopPropagation` → 触发 `scoop:apps:install`,进入 P08(不打开 P07)
- 行内"已装"按钮:`disabled`,点击无反应
- 行内"重试"按钮:`stopPropagation` → 触发重新 `scoop:apps:install`,进入 P08

### 7.4 安装中状态(§8.4)

- 点击"安装"后,该行按钮立即变 "安装中…" + Spinner + disabled
- 其他行按钮**不**受影响(每个 job 独立 P08 对话框,§8.4 + §8.12)
- P08 终态成功 → 按钮变 "已装" disabled,Toast success 3s(§8.5)
- P08 终态失败 → 按钮变 "重试" destructive,Toast destructive 8s(§8.5)
- P08 终态取消 → 按钮变 "安装"(可重试),**不展示为失败**(§8.4)

### 7.5 排序(本地)

- "按名称":字典序(zh 按 pinyin,en 按字母)
- "按更新时间":倒序(最新在前)
- 排序仅作用于当前可见结果(搜索结果或 Tab 过滤后),不重新触发 IPC

### 7.6 桶范围徽标
