## 1. 页面定位

P09 是 P03 主界面壳内的设置路由页,不是首启独立页。页面需要展示完整桌面壳上下文:左侧 240px Sidebar、右侧 PageHeader 与 PageBody。暗色是默认与主推视觉,亮色仅作为主题选项之一。

本页覆盖:

- 界面语言切换:中文 / English 两个 Radio,点击立即生效并异步持久化。
- 主题选择:亮 / 暗 / 跟随系统三个 Card,默认“暗”选中。
- 未来阶段占位:自动更新、备份还原、托盘设置、Scoop 配置、shims 管理,仅展示不可点击占位。
- 保存失败反馈:Sonner destructive Toast,包含错误码 `E_PREFERENCES_WRITE_FAILED`。

## 4. Section 1 · 界面语言

### 4.1 当前语言指示条

使用 `bg-bg-overlay` + `border-border` 的状态条。左侧 Globe 图标,中部两行文字,右侧当前语言 Pill。

中文态:

- 标题:当前语言
- 说明:下次启动将默认使用此语言
- Pill:中文 / zh-CN

英文态:

- Title:Current language
- Hint:Next launch will use this language by default
- Pill:English / en-US

### 4.2 Radio 选项

只允许两个选项:

1. 中文
   - 副文案:简体中文 · zh-CN
2. English
   - 副文案:English (US) · en-US

选中态必须满足:

- 行容器 `border-primary`
- 行背景 `bg-primary/10`
- Radio 外圈 run-green
- Radio 内圆 dot run-green
- 外环 `0 0 0 4px primary-glow`

不可出现第三语言、自动检测语言、导入语言包、运行时翻译入口。

## 5. Section 2 · 主题

三张 Card,顺序为:亮、暗、跟随系统。默认“暗”选中。

| 选项 | 中文文案 | English copy | 图标 |
| --- | --- | --- | --- |
| light | 亮 / 浅色界面 | Light / Light interface | Sun |
| dark | 暗 / 默认暗色 | Dark / Default dark | Moon |
| system | 跟随系统 / 匹配系统外观 | System / Match OS appearance | Monitor |

选中态必须使用 `bg-primary/10` + `border-primary` + micro-glow,并可显示 check 图标。未选中态保持 `bg-elevated` 与 `border-border`。

说明:API 当前 `PreferencesSchema` 明确包含 `uiLanguage`,主题字段是否入库由实现阶段按架构更新对齐;原型只表达 P09 主题入口与视觉状态。

## 6. Section 3 · 即将推出

展示五个不可交互占位项:

1. 自动更新 / Auto update
2. 备份还原 / Backup & restore
3. 托盘设置 / Tray settings
4. Scoop 配置 / Scoop configuration
5. shims 管理 / Shims manager

每个占位项:

- 背景 `bg-bg-overlay`
- 边框 `border-dashed border-border`
- `cursor-not-allowed`
- 右上角 Badge:“即将推出” / “Coming soon”
- 不设置 onClick,不作为按钮,不进入 Tab 顺序

这些项是未来阶段说明,不能暴露真实配置入口。

## 7. 场景矩阵(PRD §4)

| 场景 | 触发 | 画面状态 |
| --- | --- | --- |
| 当前查看 | 从 P03 进入设置页 | 中文 UI,中文 Radio 选中,暗色主题选中,无 Toast |
| 切到中文 | 当前为 English 时选择中文 | UI 立即变中文,中文 Radio 选中,无成功 Toast |

场景切换控件仅用于 HTML 原型演示,生产实现不需要保留。

## 8. 保存失败 Toast

使用 Sonner destructive 风格,位于窗口右上角。错误为单条偏好写入失败,不使用 Dialog,不显示原始堆栈。

中文:

- 标题:语言已切换,但保存失败
- 描述:本次切换已生效,但下次启动可能仍按旧语言显示。
- 错误码:`E_PREFERENCES_WRITE_FAILED`

English:

- Title:Language switched, but saving failed
- Description:The change is active now, but next launch may revert to the previous language.
- Error code:`E_PREFERENCES_WRITE_FAILED`

错误码必须使用 JetBrains Mono / font-mono。

## 10. API 对齐

P09 只依赖偏好读写通道:

- `scoop:prefs:get`:读取当前 `Preferences`,用于初始 `uiLanguage`。
- `scoop:prefs:set`:写入 `Partial<Preferences>`,语言切换时传入 `{ uiLanguage: 'zh-CN' | 'en-US' }`。
- 保存失败错误码使用 `E_PREFERENCES_WRITE_FAILED`。

Radio 点击后立即切换 i18n provider,再异步调用 `scoop:prefs:set`。失败时保持当前 UI 语言,只用 destructive Toast 提示“本次已生效,下次启动可能回退”。
