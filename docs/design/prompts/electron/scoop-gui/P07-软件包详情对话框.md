## 1. 页面定位

P07 是 **Dialog 而非独立路由**(设计系统 §9.3 骨架)。从 P04 已装列表行点击 / P05 浏览列表行点击进入。

- **不渲染 P03 主界面壳**(Sidebar / PageHeader)
- **窗口其余部分被遮罩 overlay**(背景 `oklch(0 0 0 / 0.5)`,与 shadcn Dialog 默认一致)
- **数据源**:`scoop:apps:info`(单包详情) + `scoop:apps:status`(是否过期,用于"已装过期"判定)
- **业务状态口径**:`docs/prd/flows/scoop-gui.md` §2.1(可装 / 已装 / 过期)

## 3. 关键组件(引用设计系统 §7)

### 3.1 DialogHeader

| 元素 | 形态 | 引用 |
| --- | --- | --- |
| Package Icon | 48×48 圆角 `rounded-md`,背景 `--muted`,内嵌 lucide `Package` 图标(24×24, 1.5px stroke, `--muted-foreground`) | §7.1/§7.3/§8.10 |
| PackageName | `text-xl font-semibold leading-tight` | §3.2"text-xl 仅 P07 软件包名" |
| Version · Bucket | `text-sm text-muted-foreground` + 分隔点(`·`),分别贴 version / source 文本 | §3.2 text-sm |
| Description | `text-sm text-muted-foreground`,1-3 行省略号(`line-clamp-3`) | §1 简洁原则 |
| Close 按钮 | 右上角,`btn-ghost icon`(lucide `X`,40×40),`focus-visible:ring-2 ring-ring ring-offset-2` | §7.1 + §8.8/§8.9 |

### 3.2 DialogBody · 两列字段组

`grid grid-cols-2 gap-x-6 gap-y-4`(对齐 §4 间距 token)。

| 字段(用户指定 6 项) | UI 类型 | 数据源(API §2.2) | 备注 |
| --- | --- | --- | --- |
| **Name** | 单行 + 复制按钮(icon button,hover 显) | `AppDetail.name` | 必填,等宽字体 `font-mono-c` |
| **Version** | 单行 + Status Tag | `AppDetail.version` | 已装时附 Tag:inactive / outdated / latest(见 §5 状态矩阵) |
| **Bucket** | 单行 | `AppDetail.source` | 等宽字体 |
| **Description** | 跨两列(`col-span-2`),多行文本 | `AppDetail.description` | 可选字段,空时隐藏整段 |
| **Homepage** | 单行外链 + 跳转 icon | `AppDetail.homepage` | URL 验证 `z.string().url()`,`target="_blank"` + `rel="noopener noreferrer"` |
| **License** | 单行 | (从 `info` 字段或 `scoop info <name>` license 段读取;AppDetail schema 未含,作为可选附加) | 缺失时显示 `—`(`text-muted-foreground`) |

每个字段用 `dl/dt/dd` 三段式或 `<div>` 区块呈现:

```html
<div>
  <div class="text-xs text-muted-foreground">Name</div>
  <div class="text-sm font-medium font-mono-c mt-1">7zip</div>
</div>
```

Status Tag(`§7` 自定义,圆角 `rounded-sm`):

| 业务状态 | variant | 颜色 token | 图标 |
| --- | --- | --- | --- |
| 可装未装 | outline | `--primary` | `Plus`(lucide) |
| 已装最新版 | secondary | `--success` 实心点 | `Check` |
| 已装过期 | destructive-soft | `--warning` | `AlertTriangle` |
| 已装失败 | destructive | `--destructive` | `XCircle` |

### 3.3 DialogBody · Tags 区(可选信息)

当满足以下条件显示:
- `AppDetail.info` 非空 → 显示"备注" Tag,内容为 info 原文(`text-xs font-medium`,`--muted` 背景)
- `outdated.manifest` 提示:已装但上游 `latest` 为空时,显示带 `--warning` 提示"Manifest 已被上游移除"

### 3.4 DialogFooter(随状态变化)

§8.2 硬约束:**右对齐,主按钮在右,次按钮(取消)在左**;不可逆操作(卸载)采用 AlertDialog 二次确认(§8.2)。

```
┌──────────────────────────────────────────────────────────────────┐
│ Footer (p-6, border-t)                                            │
│  [取消 secondary]                              [安装 default]     │   ← 可装
│  [取消 secondary]                              [卸载 destructive] │   ← 已装最新版
│  [取消 secondary] [卸载 destructive]           [更新 default]     │   ← 已装过期
│  [取消 secondary] [卸载 destructive]           [安装 default]     │   ← 已装但不支持更新(罕见,如管理员模式禁止)
└──────────────────────────────────────────────────────────────────┘
```

按钮尺寸:全部 `h-10`(§7.1 size=default),不放大。卸载/更新均为主操作级别(`h-10`),不放大到 lg。

### 3.5 AlertDialog · 卸载二次确认(§8.2)

卸载按钮点击 → 打开嵌套 AlertDialog(`max-w-md`,居中):

```
┌── AlertDialog (max-w-md) ──────────────────────┐
│  DialogHeader                                    │
│   ⚠ 卸载软件包                                    │
│   此操作不可撤销,所有应用数据将被移除。           │
│  DialogFooter                                    │
│   [返回 outline]   [确认卸载 destructive]       │
└──────────────────────────────────────────────────┘
```

- 卸载文案使用 `AlertTriangle`(lucide,`--destructive`)
- "返回"按钮关闭 AlertDialog(回到 P07);"确认卸载"触发 P08
- **不分先后版本**(防止误触):焦点默认在"返回"按钮(更安全的默认),Tab 切到"确认卸载"
- i18n key 见 §6

## 5. 状态切换矩阵(覆盖 PRD §4 全部场景 + §5 验收点)

按"按钮随状态变化"硬约束,逐项对照 PRD §4 五场景与 §5 验收:

| 场景 | PRD §4 / §5 锚点 | 触发来源 | 数据源组合 | Header 状态 | Body Tags | Footer 按钮(右对齐) |
| --- | --- | --- | --- | --- | --- | --- |
| **A. 可装未装(浏览)** | §4 场景2 / §5 ✓"可装未装提供安装入口,不应提供卸载" | P05 行点击 | `info` + 未出现在 `listInstalled` | 蓝色 Plus 图标 Tag:`可装` | 无(未装无 info 备注) | `[取消 secondary]` `[安装 default]` |
| **B. 已装最新版(列表点击)** | §4 场景1 / §5 ✓"已装提供卸载入口,不应重复安装" | P04 行点击 | `info` + 出现在 `listInstalled` 且 `latest === installed` | 绿色 Check 图标 Tag:`已装最新版` | 若 `info` 非空 → 显示 | `[取消 secondary]` `[卸载 destructive]` |
| **C. 已装过期(列表点击)** | §4 场景1 / §5 ✓"已装过期同时提供更新入口" | P04 行点击(行首有 warning 图标) | `info` + `listInstalled` + `status` 报告 `isOutdated: true` | 黄色 AlertTriangle 图标 Tag:`过期:已安装 X.X,上游 Y.Y` | 显示 `info` | `[取消 secondary]` `[卸载 destructive]` `[更新 default]` |
| **D. 已装过期 + 准备更新** | §4 场景3 / §5 ✓"触发更新进入 P08" | P07 内点更新 | 同 C | 同 C(对话框关闭进入 P08) | 同 C | 点击更新 → P08 |
| **E. 详情读取失败** | §4 场景5 / §5 ✓"详情读取失败应看到友好错误+重试" | 所有入口,`scoop:apps:info` 失败 / `null` | 错误对象 | 红色 XCircle 图标 + "无法读取软件包详情" | 无 | `[关闭 secondary]` `[重试 default]` |
| **F. 操作进行中(由 P08 覆盖)** | 设计系统 §8.4 / PRD §3.2 | 用户已触发安装/卸载/更新 | — | — | — | 全部按钮 disabled(`opacity-50 cursor-not-allowed`),关闭按钮保留可点(配合 P08 §5.4 非 modal) |
| **G. 操作不可用兜底** | §4 场景4 / §5 ✓"可装未装不展示卸载,已装最新无更新" | A 不会出现卸载;B 不会出现更新 | — | — | — | 不渲染对应按钮,而不是 disabled 灰显(§1 "简洁、不堆叠装饰元素") |

> 说明:§7 按钮随状态变化 = "只渲染可执行的操作",不渲染将灰显的按钮(避免诱导用户先点再看提示)。

## 7. 交互细节

### 7.1 加载态(进入 P07 后 ~0-1s)

`scoop:apps:info` 调用期间,Header 区显示:
- Package Icon 容器内显示 `Loader2` spin(图标本体 spin,容器圆角不变)
- 字段区全部 `bg-muted animate-pulse`(骨架屏,仅 4 行 Name/Version/Bucket/Homepage 占位,Description 隐藏)
- DialogFooter 隐藏所有操作按钮,只显示 `Loader2 + "读取中…"` 占位文案(避免在数据未齐时点击)
- 详情返回成功后,骨架屏淡出 + Footer 按钮按状态入场

PRD §5 验收点 ✅ "详情内容应来源于 Scoop 所识别的软件包信息"。

### 7.2 复制 Name(辅助操作)

Name 字段值 hover 时右侧浮现 16×16 `Copy` icon button(`btn-ghost icon`)。点击调用 `navigator.clipboard.writeText(name)` + 弹出一个 transient 3s Toast(`success` variant,`apps.detail.toast.copied`:`已复制 / Copied`)。

避免在名称值本身上做用户交互(防误触)。

### 7.3 Homepage 外链安全

```html
<a href={homepage} target="_blank" rel="noopener noreferrer"
   class="text-sm font-medium text-info-c hover:underline inline-flex items-center gap-1">
  {host(homepage)} <ExternalLink size={14} />
</a>
```

- 仅显示 host(`new URL(homepage).host`),避免长 URL 占两行
- 点击调用 `window.api.openExternal(homepage)`(由 developer 实现,经 preload 转发到主进程 `shell.openExternal`,**禁止直接 `window.open`** 防绕过 sandbox)

### 7.4 操作进行中(§8.4 硬约束)

用户在 P07 触发 install / uninstall / update 后:
1. 立即关闭 P07 dialog,主进程启动对应 job
2. 弹出 P08 进度对话框(非 modal,可与 P07/P04/P05 共存于页面上,但实际 §8.4 规定该页面其他按钮全部 disabled)
3. P07 入口(列表行)在 P08 进行中仍可被点击,但打开的 P07 中 Footer 按钮全 disabled,提示"正在 {installing/updating/uninstalling} ..."

### 7.5 卸载二次确认(`/uninstall` 不可逆硬约束)

- 点击 Footer `[卸载]` → 弹 AlertDialog(nested)
- 用户在 AlertDialog 按 Enter 默认焦点在"返回"按钮(更安全的默认,§3.5)
- AlertDialog 内 Esc 关闭等价于"返回"
- 离开 AlertDialog 后用户可在 P07 重复触发(不强制冷却期)

### 7.6 失败重试(场景 E / PRD §5 ✓)

E 状态下 Footer:
- `[关闭 secondary]`(左):关闭对话框,回 P04/P05 列表
- `[重试 default]`(右):重新调用 `scoop:apps:info`,重复 7.1 加载态

### 7.7 关闭行为统一

- Header 右上角 X、Footer 取消、Esc、点击遮罩(任一)→ 关闭 P07,返回 P04/P05,**不改动源列表数据**
- 操作成功后(从 P08 终态 success 收到事件)→ 自动关闭 P07 + 触发来源页 P04/P05 刷新(由 developer 在 P08 hook 里调用列表 invalidate 逻辑)
