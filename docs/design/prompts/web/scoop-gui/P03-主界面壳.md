> 文档层级:页面级产出(designer · 仅登记不送审)
> 适用范围:web 端 scoop-gui 模块 P03

## 1. 设计目标与页面定位

P03 是产品进入主流程后的唯一入口壳层:承载侧边导航(Sidebar)与主内容区(Main),自身不持有任何具体业务数据,只负责"导航 + 路由到 P04/P05/P06/P09"与几项横切能力的呈现容器(F18 单实例聚焦后的落点、F14 语言切换的入口、F19 友好提示的承载区)。

P03 没有独立于设计系统之外的视觉主题——它的全部美学都来自设计系统 v2 的通用 token 与组件形态(`docs/design/systems/web.md` 第 2~9 节)。本文档只回答"这些 token 如何组装成 P03 特有的骨架",不重复定义颜色/字体/间距本身。

## 2. 整体骨架(严格对应设计系统 §10.1)

```
┌ Sidebar 顶部 1px 渐变线(仅贯穿 240px 宽度,不跨越整窗)──────────────┐
│ [品牌区 LOGO + 产品名 + scoop 版本]  │ [内容区顶部细渐变条 2px]        │
│ [4 个导航项:已装软件/浏览软件包/桶管理/设置] │ [PageHeader:标题+描述+占位工具栏] │
│ (弹性留白,把底部块推到最下)         ├──────────────────────────────────┤
│ ── 分隔线 ──                        │                                  │
│ [Theme 三态切换:暗/亮/跟随系统]      │   主内容区(bg-grid + bg-glow-top)│
│ [语言二态切换:中文/EN]               │   当前为占位:提示"实际内容由     │
│                                      │   P04/P05/P06/P09 承载"          │
└──────────────────────────────────────┴──────────────────────────────────┘
```

- Sidebar 固定 240px,不随窗口缩放变化;内容区弹性伸展(`flex:1`)。
- 整体窗口默认 1280×800,最小 1024×600,支持 resize(与 `src-tauri/tauri.conf.json` 实际配置一致,亦与设计系统 §9 一致,无需在原型里额外强调,但原型的最小宽度应设置为 1024px 以便预览时校验断点)。
- 本骨架**不包含**自绘窗口标题栏(最小化/最大化/关闭按钮)——`tauri.conf.json` 未设置 `decorations:false`,窗口使用系统原生标题栏,原型不应画蛇添足加一条假标题栏。

## 3. 侧边栏(Sidebar)规格

| 属性 | 规格 |
|---|---|
| 宽度 | 240px,固定,不参与 flex 伸缩 |
| 背景 | `--card`(暗色 `#111827`) |
| 右边框 | `1px solid var(--border)` |
| 顶部装饰 | 1px 渐变线,`linear-gradient(to right, transparent, var(--primary), transparent)`,宽度=侧边栏宽度(240px),不是整窗宽度(见第 6 节说明) |
| 内部纵向结构 | 品牌区 → 导航区 → 弹性留白 → 分隔线 → 底部功能区 |
| 底部内边距 | 与顶部品牌区内边距对称,建议 `padding-bottom: 12px` |

## 4. 品牌区(Logo)规格

- 图标容器:36×36px,圆角 `--radius-md`,边框 `1px solid var(--primary-glow)`,背景 `var(--primary-glow-soft)`,内部为 Terminal/命令行图标(`text-primary`,呼应"终端原生感"设计原则)。
- 产品名:`Scoop GUI`,字体 `--font-heading`(Space Grotesk),字重 700,字号 15px。
- 副标题:当前 scoop 版本号占位(如 `scoop 0.5.3`),字体 `--font-mono`,10px,`text-subtle`,单行省略(`overflow:hidden;text-overflow:ellipsis;white-space:nowrap`,避免长路径/版本号撑破 240px)。
- 品牌区与导航区之间留白:约 14px。

## 5. 导航项规格(4 项)

| 顺序 | key | 中文文案 | English | 图标语义 | 对应出口页面 |
|---|---|---|---|---|---|
| 1 | installed | 已装软件 | Installed | 包裹盒(package) | P04 已装软件包列表页 |
| 2 | browse | 浏览软件包 | Browse | 放大镜(search) | P05 浏览/搜索软件包页 |
| 3 | buckets | 桶管理 | Buckets | 层叠图层(layers) | P06 桶管理页 |
| 4 | settings | 设置 | Settings | 滑块组(sliders) | P09 设置/语言切换页 |

> 文案取自 `src/i18n.ts` 的 `nav.installed` / `nav.browse` / `nav.buckets` / `nav.settings` 实际字典值,而非页面 PRD 中的简写"已装/浏览/桶/设置"——原型与后续实现应以字典原文为准。

**尺寸与状态**:

- 单项:高 40px(`h-10`),水平内边距 12px(`px-3`),图标与文案间距 10px,圆角 `--radius-md`。
- 图标尺寸 16px,文案 13px / DM Sans 500。
- **默认态**:文案 `text-muted-foreground`,透明背景。
- **hover 态**(非选中):背景 `--secondary`,文案 `--foreground`。
- **选中态**:背景 `var(--primary-glow-soft)`,文案 `text-primary`,左侧内嵌 2px `--primary` 边线(`box-shadow: inset 2px 0 0 var(--primary)`,而非真实 border,避免挤压布局宽度)——与设计系统 §8.6 表格选中行使用同一视觉语言(主色 10% 透明背景 + 左侧 2px 边线),形成"选中态"跨组件一致性。
- **同一时刻只有一项处于选中态**,与 `view` 状态一一对应。
- **右侧可选徽标**:当某导航项存在可提示的计数(如"已装软件"下有过期项)时,使用 `Badge` 的 `warning` 变体(`TriangleAlert` 图标 + 数字),右对齐,不挤压文案(文案区 `flex:1` 且 `overflow:hidden`)。原型中以静态示例数字演示该形态,标注为"示例数据,非实时"。
- **键盘/无障碍**:每项是真实 `<button>`;`aria-current="page"` 标记选中项;`:focus-visible` 使用与 Button 组件一致的聚焦环(`box-shadow: 0 0 0 3px var(--primary-glow), 0 0 0 1px var(--primary)`);Tab 顺序 = 视觉顺序(品牌区不参与 Tab,4 个导航项依次,再到底部 Theme/语言控件)。

## 6. 品牌线与内容区顶部渐变条(两条不同的线,不要混淆)

设计系统 §10.1 的 ASCII 骨架里出现了两条视觉上都是"渐变线"的元素,但语义、位置、宽度都不同,原型必须分开实现:

1. **Sidebar 顶部品牌线**(设计系统 §9 硬约束"顶部品牌线"条目明确锚定的那一条):1px 高,`from-transparent via-primary to-transparent`,宽度 = Sidebar 宽度(240px),位于 Sidebar 最顶部、品牌区之上。这是纯装饰性的"科技感"点缀,常驻显示,不随任何状态变化。
2. **内容区顶部细渐变条**:2px 高,位于内容区最顶部、PageHeader 之上,横跨整个内容区宽度(不含 Sidebar),配色 `primary → info → primary`(即 P08 进度对话框 `.running-bar` 使用的同一组配色,见设计系统 §7 / §10.4)。

关于第 2 条渐变条是否应"常驻显示"还是"仅在有任务执行中时显示",设计系统 §10.1 骨架图本身把它画在了页面骨架里(暗示常驻),而 §10.4 又明确把同款渐变条定义为 P08 的"进行中"专属指示。本原型按**常驻显示但不带动画**处理(静态渐变,不加 `background-position` 位移动画),仅作为 P03 壳层的品牌点缀;若 developer 希望呼应"当前有任务在后台执行"这一全局状态,可将其替换为条件渲染的 `.running-bar`(仅当 `activeJobCount > 0` 时出现动画),两种做法都不违反设计系统,本文档记录该分歧点供决策留痕,不代为决策。

## 7. 侧边栏底部:Theme toggle 与语言切换

严格按设计系统 §8.7("底部 Theme toggle(亮/暗/跟随系统)")与 §10.1 骨架实现:

- 弹性留白(`flex:1`)把以下内容推到 Sidebar 最底部。
- 分隔线:`1px solid var(--border)`,上下各留 8px 间距。
- **Theme 三态切换**:一个分段控件(segmented control),三个等宽分段——暗(默认选中,月亮图标)/ 亮(太阳图标)/ 跟随系统(显示器图标)。选中分段:`bg-[var(--primary-glow-soft)] text-primary`;未选中:`text-muted-foreground`,hover 时 `bg-secondary`。容器:`border border-border rounded-md`,高度 32px。
- **语言二态切换**:同款分段控件视觉语言,两个分段——中文 / EN。默认选中"中文"(与产品首次启动后未做选择前的引导态无关,这里只是 Sidebar 常驻控件的默认展示值)。
- 两组控件之间垂直间距 10px;每组控件上方各有一行 11px 大小写字母间距的小标题("主题" / "语言"),`text-subtle`。

## 8. 内容区占位规格(PageHeader + 空状态承载说明)

- **PageHeader**:左侧为标题(`text-2xl font-semibold`,随当前选中导航项变化)+ 一行描述(`text-[13px] text-muted-foreground`);右侧为一个禁用态的"更多操作"图标按钮占位(对应 §10.1 ASCII 中的"⋯"),提示"具体工具栏由各子页面自行定义"。PageHeader 底部有 `1px solid var(--border)` 分隔线。
- **主内容区**:铺满剩余高度,叠加 `.bg-grid`(32px 网格线,8% 不透明度)与 `.bg-glow-top`(顶部 primary 微光径向渐变),与设计系统 §9"网格背景"硬约束("主页 P03/P04/P05/P06/P08/P09 顶部 1/3 区域叠加")一致。
- 内容区中央放置一段占位说明(图标 + 标题 + 一行说明文字),明确告知"此内容区在生产环境由 P04/P05/P06/P09 承载,当前仅为 P03 壳层原型演示"。切换左侧导航项时,占位标题与说明文字应随之更新(演示"主区随导航切换"这一核心交互,而不需要真的实现四个子页面)。

## 9. 设计 token 命名对照(实现口径说明)

设计系统 §11.1 给出的 `@theme` 示例变量名(如 `--color-bg`、`--color-fg-muted`)是**说明性命名**,而项目实际的 `src/index.css` 落地时改用了 shadcn 语义化命名(`--background`、`--muted-foreground` 等),并把 §11.1 的"accent"(信息蓝 `#60A5FA`)与真实 CSS 里的 `--accent`(shadcn 语义槽位,实际是中性 hover 背景色,值与 `--secondary` 相同)拆成了两个不同的变量——真正的信息蓝在真实实现里叫 `--info`。本原型的 CSS 直接采用 `src/index.css` 的真实变量名与取值(而非设计系统 §11.1 的示例命名),使原型可与生产 CSS 逐项对照。对照表:

| 设计系统 §11.1 示例命名 | `src/index.css` 实际命名 | 暗色取值 |
|---|---|---|
| `--color-bg` | `--background` | `#0b1220` |
| `--color-bg-elevated` | `--card` | `#111827` |
| `--color-bg-overlay` | `--secondary` / `--muted` | `#1a2332` |
| `--color-fg` | `--foreground` | `#f5f7fa` |
| `--color-fg-muted` | `--muted-foreground` | `#a1aab8` |
| `--color-fg-subtle` | （无 `--color-` 前缀)`--subtle` | `#6b7280` |
| `--color-border` | `--border` | `#374151` |
| `--color-border-strong` | `--border-strong` | `#5b6678` |
| `--color-primary` | `--primary` | `#22c55e` |
| `--color-accent`(信息蓝) | `--info` | `#60a5fa` |
| （shadcn 语义槽,非设计系统 §11.1 概念) | `--accent`(hover 背景,非信息蓝) | `#1a2332` |
| `--color-warning` | `--warning` | `#f59e0b` |
| `--color-destructive` | `--destructive` | `#ef4444` |

## 10. 与当前 `src/App.tsx` 实现的已知差异(供 developer 核对,不构成本文档范围变更)

本节记录本原型严格遵循设计系统 §8.7/§10.1 骨架之后,与当前已上线的 `src/App.tsx` 之间存在的结构性差异。这些差异不在本设计提示词的职责范围内裁决(裁决权属于共享契约文档的统一维护者),仅作登记:

1. **Sidebar 底部内容不同**:设计系统 §8.7/§10.1 要求 Sidebar 底部为"Theme toggle + 中英切换"两组常驻控件(本原型即按此实现);而 `src/App.tsx` 的 `Shell` 组件在同一视觉位置(弹性留白之后)渲染的是一个"任务"(Jobs)导航项——点击后展开 `JobsPanel`(P08)浮层,并带有进行中任务数量的徽标。真实实现里,Theme 切换与语言切换并不在 Sidebar 出现,而是被收纳进 P09 设置页(`src/pages/SettingsView.tsx`)。
2. **P03 页面 PRD 与真实实现在这一点上互相自洽**:页面 PRD `docs/prd/pages/web/scoop-gui/P03-主界面壳.md` 对 F14 的表述是"提供进入 P09 的设置入口"(而非"侧边栏直接提供切换控件"),这与 `src/App.tsx` 的真实做法一致;**产生分歧的一方是设计系统文档本身**(§8.7/§10.1 描述的是"直接内嵌"的形态)。
3. 本原型按任务方要求"严格按 §10.1 骨架"产出,因此仍然画出了 Theme toggle + 中英切换;是否需要把 Sidebar 骨架改回"任务入口"或者反过来把真实实现改造成"侧边栏直接切换 Theme/语言",由后续统一裁决,不在本文档内展开选型讨论。

## 11. 第 9 节硬约束逐条勾稽(仅列与 P03 相关的行)

| 硬约束 | 本原型落实方式 |
|---|---|
| 窗口尺寸 1280×800 默认 / 最小 1024×600 | 原型 `body` 设置 `min-width:1024px; min-height:600px`,并在注释中说明生产窗口默认尺寸 |

| i18n:两套双语占位,禁止只有一种语言的文案 | 全部可见文案均以 `.zh`/`.en` 双 span 实现,语言切换控件可实时验证 |

| 网格背景:P03 顶部 1/3 区域叠加 `.bg-grid`+`.bg-glow-top` | 内容占位区域实现,详见第 8 节 |
| 顶部品牌线 | Sidebar 顶部 1px 渐变线,详见第 6 节 |
