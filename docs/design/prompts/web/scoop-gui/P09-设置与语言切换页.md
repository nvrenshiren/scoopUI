## 1. 页面定位

P09 不是首启独立页,而是 P03 主界面壳内的一个路由态(`view: "settings"`),由侧边栏"设置"导航项切换进入,与"已装/浏览/桶管理"三个视图对等地承载于同一个 `<main>` 容器内——**不是** Dialog/Modal。设计时必须呈现完整壳层上下文(240px Sidebar + 主区),不能把 P09 单独抽出为一个无壳的独立画面。

页面职责严格对应 F14(界面语言切换)+ 设计系统 §9 硬约束"P09 提供亮/暗/跟随系统三选一",外加两块只读信息回显(上次协助安装配置、关于)。不引入 Scoop 配置编辑、备份还原、托盘设置等阶段二能力的任何入口(哪怕是禁用占位)。

## 3. Section 1 · 界面语言(F14)

- 图标:`Globe`。标题:"界面语言"/"Language"。说明:"切换后立即生效,并保存在本机"/"Takes effect immediately and is saved on this machine"。
- 控件:两个并排按钮,**顺序固定为"中文"在前、"English"在后**,不随当前语言改变位置。
- 按钮标签本身是语言自称名("中文"“English”),**不经过 i18n 字典翻译**——无论当前界面语言是中文还是英文,这两个标签文字始终不变,这是语言选择器的正确惯例(自称名不翻译),设计与实现都不应该给它们接入 `t()`。
- 选中态:`variant=default`(实心 run-green + `box-shadow: 0_0_12px_var(--primary-glow)`,hover 增强光晕)且左侧带 14px `Check` 图标;未选中态:`variant=outline`(透明背景 + `border-border`,hover 背景变 `--secondary`),不带图标。
- 点击已选中的选项应为静默 no-op(不重复触发保存动作)。
- 点击未选中的选项:①立即让当前页面与整个应用(侧边导航、任意已打开视图)文案切换为目标语言;②异步持久化;③不阻塞、不显示 loading 态;④若持久化失败,只弹出 **info 级** Toast(见第 7 节),不改变已切换的界面文案,不判定为失败态。
- 语言切换不得触发或影响任何软件包 / 桶 / InstallJob 的业务状态(与安装、卸载、更新、桶增删完全解耦)。

## 4. Section 2 · 主题(暗 / 亮 / 跟随系统)

- 图标:`SlidersVertical`。标题:"主题"/"Theme"。**无**说明文字(与 Section 1 不同,不要补一条 desc)。
- 控件:三个并排按钮,固定顺序"暗色→亮色→跟随系统"("dark" → "light" → "system"),按钮形态、选中态/未选中态视觉规则与 Section 1 完全一致(同一套 Button 组件、同一套 Check 图标用法)——**不使用**独立的"主题预览卡片"或每个选项专属图标(不加 Sun / Moon / Monitor 小图标),保持与语言按钮同构,不额外发明视觉层级。
- 默认态:本机未持久化过主题时,视觉呈现应为"暗色"选中(即使 `Settings.theme` 为空,产品在启动时也会以 `dark` 兜底应用)。
- 点击后：①立即切换全局视觉主题(`light` 态在 `<html>` 挂 `.light` 类,变量表见设计系统 §3/§11.1 的亮色覆盖);②"跟随系统"态需要绑定一次 `matchMedia('(prefers-color-scheme: dark)')` 的 change 监听,保证操作系统主题切换时界面能continue 联动更新,而不仅仅是点击瞬间生效一次;③异步持久化,失败时与语言切换同规格的 info Toast。
- 主题切换同样不得影响任何业务实体状态。

## 5. Section 3 · 上次确认的 Scoop 安装配置(只读回显)

- 图标:`Download`。标题:"上次确认的 Scoop 安装配置"/"Last confirmed Scoop install options"。说明:"协助安装 Scoop 时确认过的配置,重装时自动带入"/"Options confirmed during assisted install; pre-filled on reinstall"。
- 这是 **F16 协助安装** 结果的只读回显,不是新的可编辑表单,不提供任何输入控件或"重新配置"按钮。
- 空态(本机 Scoop 并非由本产品协助安装,即 `installConfig` 未持久化):展示一行 `text-subtle text-[13px]` 提示——"尚未通过本产品安装过 Scoop"/"Scoop has not been installed through this app"。
- 有值态:两列 `<dl>`(`grid-cols-[240px_1fr]`,`font-mono text-xs`),仅列出"有值"的字段,固定顺序为 `ScoopDir → ScoopGlobalDir → ScoopCacheDir → Proxy → NoProxy → ProxyUseDefaultCredentials → RunAsAdmin`;布尔字段有值时显示字面 `"true"`。
- 字段标签使用与 `install.ps1` 官方参数一致的英文 PascalCase 原名(`ScoopDir`、`RunAsAdmin`…),**不做中英文翻译**,不套用 `detail.field.*` 那类 i18n 回退规则——这是刻意的对照展示,不要在设计稿或视觉规范里给这些标签编造中文译名。
- **严禁**展示代理凭据的用户名 / 密码字段(`ProxyCredential`/密码),即使数据结构里存在这两个字段。回显区只读且允许 `select-text`(可选中复制路径等值),但不得让敏感凭据以任何形式出现在这块只读展示里。

## 6. Section 4 · 关于

- 图标:`Info`。标题:"关于"/"About"。无独立说明文字。
- 两行只读 `<dl>`(格式同 Section 3):
  - "应用版本"/"App version" → 固定字面量 `Scoop GUI 0.1.0`(当前在源码中是硬编码字符串,不随构建版本号动态生成,设计稿按此现状展示,不擅自加"自动读取 package.json"之类的动效说明)。
  - "Scoop 版本"/"Scoop version" → 运行期探测到的本机 Scoop 版本号;未探测到时用全角破折号 `—` 占位。

## 7. 交互与场景矩阵(对齐页面 PRD §4)

| 场景 | 触发 | 页面应呈现 |
| --- | --- | --- |
| 场景 1·当前查看 | 从 P03 设置入口进入 | 当前语言按钮与当前主题按钮均为选中态(实心+Check),其余两块只读区按当前本机数据展示 |
| 场景 2·切到中文 | 当前为 English 时点击"中文" | 设置页与侧边导航、已打开视图文案立即变为中文;"中文"按钮变为选中态;不出现任何 Toast |
| 场景 3·切到 English | 当前为中文时点击"English" | 同上,方向相反;"English"按钮变为选中态;不出现任何 Toast |
| 场景 4·保存失败 | 语言或主题切换后,本机持久化失败 | 界面已按用户选择即时呈现(不回滚);右上角出现 **info 级**(非 destructive/错误红)Toast:"设置已生效,但保存到本机失败;下次启动可能恢复旧值"/"Applied, but saving failed — the old value may return on next launch" |

四个场景与页面 PRD 第 4 节交互说明的"场景 1~4"一一对应;场景切换控件仅用于 HTML 原型演示自检,不代表生产实现需要保留。

## 8. 与上游文档的口径差异(据实登记,不改写上游文件)

以下差异以 `src/pages/SettingsView.tsx`、`src/store.ts`、`src/App.tsx` 的真实实现为准;上游文档如需更新,应另行走评审流程,本提示词只负责如实登记,不代为修改 PRD / 设计系统原文。

1. **主题能力在 P09 页面 PRD 中完全缺席**:`docs/prd/pages/web/scoop-gui/P09-设置与语言切换页.md` 的"功能清单"只登记了 F14 + F19,全文没有一处提到主题/亮暗切换;但设计系统 `docs/design/systems/web.md` §9 硬约束明确写着"P09 提供'亮/暗/跟随系统'三选一",且 `SettingsView.tsx` 确实实现了完整的主题三选一(见第 4 节)。本提示词按"设计系统硬约束 + 真实代码"两者一致的口径,把主题三选一作为 P09 的正式组成部分设计,而非 PRD 字面描述的"仅语言切换"。
2. **"Sidebar 底部 Theme toggle"在真实代码中不存在**:设计系统 §8.7("底部 Theme toggle(亮/暗/跟随系统)")与 §10.1 页面骨架示意图(`[Theme]` / `[中文/EN]` 均画在 Sidebar 底部)都描述了一个 Sidebar 内嵌的主题/语言快捷切换区;但 `src/App.tsx` 的 `Shell` 组件里,Sidebar 底部只有一个"任务"导航项,没有任何主题或语言控件——主题与语言的唯一切换入口是 P09 本页。设计提示词按真实壳层结构处理,不在 Sidebar 内补画一个代码中不存在的控件。
3. **持久化失败反馈的级别**:旧版(Tauri 迁移前)Electron 设计稿曾把该提示设计为 destructive(错误红 + 微光)Toast,并配错误码 `E_PREFERENCES_WRITE_FAILED`。当前真实实现里 `switchLanguage`/`switchTheme`(`src/store.ts`)在持久化失败时统一调用 `toast.info(...)`,对应 `docs/prd/flows/scoop-gui.md` §1.8 异常分支"该情形仅以业务语言告知用户,本流程不进入失败态"的业务规则——即时生效与持久化失败是两件独立的事,不应被当作错误处理。本提示词按 info 级别设计,不沿用旧稿的 destructive 处理,也不再使用已经不存在的错误码体系(当前 Tauri 命令失败时返回的是普通字符串错误,如 `unsupported language`/`unsupported theme`,不是结构化错误码)。
4. **只读信息区在页面 PRD 中未被提及**:页面 PRD 全文没有"上次确认的安装配置"或"关于"这两块内容,但它们是 `SettingsView.tsx` 中真实存在、且始终渲染的两个 Section。按"以代码为准"的原则,本提示词把它们作为 P09 的正式组成部分登记,而不是遗漏。
5. **未采用旧版"即将推出"占位区**:Tauri 迁移前的 Electron 版设计稿曾包含一个"即将推出"Section(自动更新 / 备份还原 / 托盘设置 / Scoop 配置 / shims 管理五个禁用占位卡)。当前真实代码里没有这块内容,且页面 PRD 验收要点明确写着"不应出现 Scoop 配置编辑等未来能力入口"、模块 PRD 决策 D01 也把这些能力排除在本阶段之外。本提示词延续现状,不恢复这块占位区。
