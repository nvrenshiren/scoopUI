## 1. 设计目标

P08 不是一个"单任务进度弹窗",而是产品内**唯一的、跨页面共享的任务观察面板**:承载 F05/F06/F07/F08/F11/F12 触发的全部 InstallJob(排队中/执行中/已成功/已失败(可重试)/已取消五态),让用户在不打断当前浏览的前提下,随时确认"我刚才发起的操作现在怎么样了"。设计基调延续设计系统 v2 的"终端原生感 + 高级感 + 科技感":玻璃质感浮层、run-green 微光、JetBrains Mono 日志区。

## 2. 与设计系统骨架图 / 页面 PRD 的偏离说明(先读,避免下一轮回归)

设计系统 `docs/design/systems/web.md` §8.5 与 §10.4 描绘的是一个"居中单任务对话框"骨架(标题栏 + 单一进度条 + 单一日志区 + 底部"关闭 ghost / 取消 destructive"两个按钮)。真实实现 `JobsPanel.tsx` 是一个**右下角悬浮的任务列表面板**,与骨架图在下列 5 点上存在实质性差异。**本提示词与随附原型按代码真实形态设计**,以下差异已按核心原则记录,不修改设计系统文件本身:

| # | 设计系统 §8.5/§10.4 描述 | 代码真实实现 | 本提示词采用 |
| --- | --- | --- | --- |
| 1 | "非 modal · **居中**浮层" | `fixed right-6 bottom-14`,**右下角**悬浮,宽 480px | 右下角悬浮 |
| 2 | 单任务:标题 + 副标题("当前步骤") + 一条进度条 + 一个日志区 | **任务列表区**(可含多条排队中/执行中/终态任务)+ **单个详情/日志区**(展示当前选中任务) | 列表 + 详情双区 |
| 3 | 进度条示例为确定值百分比(`45%`) | `Progress` 组件 `value` 恒为 `null`(Scoop CLI 无结构化进度输出),**恒为不确定态扫动动画** | 只做不确定态 |
| 4 | 底部固定一行"关闭 ghost + 取消 destructive(仅 running 时)" | 关闭是面板**头部右上角 X**(ghost、iconSm),任何时刻可点;取消是**每个排队中/执行中任务行内联的小按钮**(可同时存在多个) | 头部 X + 行内取消 |
| 5 | 取消态色板:8.5 正文写"禁用 destructive,改用 `--fg-muted`",但 10.4 骨架图自身却把取消按钮标注为"取消 destructive" | 代码 `variant="cancel"` 明确使用中性灰(`bg-secondary` + `text-muted-foreground`),**不是** destructive | 中性灰(设计系统 §8.5 正文与 §9 硬约束口径,10.4 骨架图的 "destructive" 标注视为该图自身的笔误,不采用) |

此外还有两点补充差异,同样按代码为准:

- **无背景遮罩**:设计系统 §8.5 写"背景半透明(`bg-black/30 backdrop-blur-sm`)",但代码里面板本身没有任何遮罩层——底层页面完全不变暗、可正常交互;"非 modal" 是通过"点击面板外部即关闭"实现的,而不是靠半透明遮罩降低底层可交互性。
- **`bg-grid`/`bg-glow-top` 归属**:设计系统 §9 硬约束表格把 "P08" 列入需要 `bg-grid + bg-glow-top` 装饰的页面清单,但这两个类实际只出现在 `InstalledView.tsx`/`BrowseView.tsx`/`BucketsView.tsx`/`SettingsView.tsx`/`BootView.tsx`/`LanguagePick.tsx` 里,`JobsPanel.tsx` 自身不带这两个类。即该装饰属于 P08 浮层背后的宿主页面(P04/P05/P06/P07 所在页面),不是 P08 浮层卡片自身的样式。本提示词与原型按此口径:浮层卡片本身不叠加网格/辉光背景,只保留其 `surface-glass` 玻璃质感。

页面 PRD(`docs/prd/pages/web/scoop-gui/P08-长时操作进度对话框.md`)第 3.1 节写"本页面由 P01 / P04 / P05 / P06 / P07 触发长时动作时进入",第 5 节验收要点也把"失败时应看到...重试入口"列为通用要求。代码层面有两处需要设计者知悉:

- **F16(协助安装 Scoop,P01 场景)不经过本组件**:`store.ts` 的 `startInstallScoop()` 只设置 `boot: "installing"`,从不设置 `jobsPanelOpen`;P01 的进度/日志/取消 UI 是 `BootView.tsx` 内部**自行绘制的另一套居中卡片**(同样用了 `Progress`/`log-box`/`cancel` 按钮,但不是 `JobsPanel` 组件,布局也是居中卡片而非右下角悬浮)。本提示词与原型**只覆盖 `JobsPanel.tsx` 实际承载的 F05/F06/F07/F08/F11/F12 六类场景**,不重画 P01 自己的安装进度卡片(那属于 P01 页面的设计范畴)。
- **面板本身不提供"重试"按钮**:失败任务在列表行里只显示"已失败(可重试)"徽标,没有任何内联重试操作;"重试"是用户返回来源页(P04/P05/P06)对同一目标**重新发起一次同类动作**(会创建一条全新 InstallJob,而不是复用旧任务)。设计与原型均按此真实路径呈现,不臆造一个代码里不存在的重试按钮。

## 3. 整体布局与定位

| 属性 | 规格 |
| --- | --- |
| 定位 | 相对于应用窗口 `position: fixed`(原型中相对窗口帧 `position: absolute`),`right: 24px`,`bottom: 56px` |
| 尺寸 | 宽 480px(不超过视口 `calc(100% - 48px)`);高度自适应内容,`max-height: min(560px, calc(100% - 120px))` |
| 层级 | `z-index: 40`,浮于主内容之上,低于 Radix Dialog / Sonner Toast |
| 背景 | `surface-glass`(半透明玻璃 + `backdrop-filter: blur(12px)`),圆角 `--radius-xl`(16px),边框 `1px solid --border`,阴影 `--shadow-xl` |
| 遮罩 | 无。底层页面保持完全可交互与正常亮度 |
| 显隐 | 面板整体只有"开/关"两态,无 modal 意义上的入场/退场遮罩动画要求;内容随任务数组增删/状态跃迁**原地更新**,不重新挂载整个面板 |
| 顶部进度条 | 当且仅当存在至少一条 `state === "running"` 的任务时,面板最顶端渲染 2px 高的 `running-bar`(`from-primary via-info to-primary` 渐变,`background-size:200% 100%`,1.6s 线性无限循环位移动画);无任务执行中时不渲染此条(不是渲染后隐藏,是不占位) |

## 4. 顶部区(Header)

- 结构:`[List 图标 14px] [标题文案 "任务进度"/"Tasks",font-heading 14px semibold] [flex 占位] [关闭按钮]`
- 关闭按钮:`ghost` variant,`iconSm`(32×32),图标为 X(14px);点击**始终**把面板状态置为关闭,不受"是否有任务执行中"影响——这是"非 modal"语义的关键体现:用户可以在任务执行中随时把面板收起去做别的事,任务不会因为面板关闭而中断或取消。
- 头部高度约 44px(`py-3` + 内容),左右内边距 16px/12px(右侧给关闭按钮留出更贴边的点击热区)。

## 5. 任务列表区

- 容器:`max-height: 180px`,超出滚动;左右内边距 8px,行间距 2px。
- 空状态:当任务集合为空时,整个列表区与详情区都不渲染,改为居中显示一行文案(`jobs.empty` = "暂无任务"/"No tasks yet"),内边距 24px,字号 13px,颜色 `--fg-muted`。
- 排序:按任务 `id` **降序**(最新创建的任务排最上面)。这与"先进先出"的实际执行顺序(单 worker、按创建顺序执行,见 `ARCH05`)是两回事——列表展示顺序只反映"谁是最近发起的",不代表执行队列顺序。设计与原型均需保留这个"展示顺序 ≠ 执行顺序"的真实特性,不要为了"看起来更符合直觉"而改成按执行顺序排序。
- 单行结构(高度约 32px,`px-2 py-1.5`,圆角 `--radius-md`):

  ```
  [状态图标 14px] [任务标题,13px,超宽省略号截断 max-w:200px] [flex 占位] [状态徽标 Badge] [取消按钮(条件渲染)]
  ```

  - 任务标题 = `<动作中文/英文标签> <目标名>`(如"安装 git"、"更新 nodejs"、"添加桶 extras"),中间只有一个空格,**没有** "scoop install git" 这种夹带命令名的写法,也没有 "·" 分隔符——直接对应 `jobTitle()` 的拼接方式。
  - 状态图标与徽标映射(照抄 `stateBadgeVariant`/`StateGlyph` 的映射关系,不要发挥):

    | 任务状态 | 图标 | 图标色 | 徽标 variant | 徽标文案(zh/en) |
    | --- | --- | --- | --- | --- |
    | 排队中 queued | List(三横线) | `--fg-subtle` | default(中性灰) | 排队中 / Queued |
    | 执行中 running | LoaderCircle(旋转) | `--primary`,持续旋转动画 | info(蓝) | 执行中 / Running |
    | 已成功 succeeded | Check | `--primary` | success(绿) | 已成功 / Succeeded |
    | 已失败(可重试) failed | TriangleAlert | `--destructive` | destructive(红) | 已失败(可重试) / Failed (retryable) |
    | 已取消 cancelled | X | `--fg-subtle` | default(中性灰,**与 queued 同一 variant**) | 已取消 / Cancelled |

    注意"已取消"与"排队中"共用同一个中性灰徽标样式(代码里两者都落入 `default` 分支),这不是遗漏,设计与原型都应保留这一"重复但故意"的映射,不要给 cancelled 单独发明一个新配色。
  - 取消按钮:**仅当该行状态为 `queued` 或 `running` 时渲染**,`cancel` variant(中性灰:`bg-secondary` + `border-border` + `text-muted-foreground`,hover 时边框转 `border-strong`、文字转前景色),尺寸 `h-6 px-2 text-xs`(比常规按钮更小,嵌在行内不抢视觉焦点)。**绝对不用 destructive 红色**——取消是用户主动终止,不是失败,这是设计系统 §9 硬约束,也是本页面唯一的强制色彩规则。
  - 选中态(该行是当前详情区展示的对象):背景 `--primary-glow-soft`,左侧内嵌 2px `--primary` 竖线(`box-shadow: inset 2px 0 0 var(--primary)`);未选中时 hover 背景为 `--bg-overlay`(等价 `--secondary`)。
  - 点击行(非点击取消按钮时)只做一件事:把该行设为详情区当前展示对象,不做其它跳转或副作用。

## 6. 详情 / 日志区

- 位于列表区下方,`border-top: 1px solid --border` 分隔,内边距 `16px 16px 15px`(左右 16、下 15、上约 10)。
- **当前展示对象的选择逻辑**(必须精确复刻,不能简化成"始终显示第一条"或"始终显示运行中那条"):
  1. 若用户已手动点击过某一行(存在"已选定" id 且该任务仍在集合中),显示该任务,不管它是什么状态、是否还在执行——即"钉住"用户的选择。
  2. 否则(用户尚未手动选择过,或钉住的任务已从集合移除),自动显示**当前状态为"执行中"的第一条任务**。
  3. 都没有的话,回退显示"按 id 降序排序后的第一条"(即列表最上面那条)。
  - 这套"未手动选择时自动跟随执行中任务"的行为需要在原型里用可交互的方式演示出来(见原型第 6 节要求),让评审者能直观看到"用户没管它时,详情区会自动切到正在跑的那个任务"。
- 标题行:`"任务日志" · <当前对象标题,font-mono>`,12px,颜色 `--fg-subtle`。
- 不确定进度条:**仅当**当前对象状态为 `queued` 或 `running` 时渲染;`succeeded`/`failed`/`cancelled` 状态下不渲染进度条(不是渲染成 100% 或灰色,而是整个进度条元素消失)。进度条本身:轨道 `--bg-overlay`,高 8px,圆角 full;填充色块宽度固定 32%,`box-shadow: 0 0 12px var(--primary-glow)`,从 `-32%` 平移到 `100%` 做 1.4s 缓入缓出无限循环扫动——**没有数值百分比**,不要画成"45%"这种确定值进度条,Scoop CLI 不产出可解析的进度百分比,产品也没有编造一个。
- 日志区:高度约 180~260px(原型可用 h-64=256px 一类相近尺寸),`overflow: auto`,背景 `rgba(0,0,0,.35)`(暗色下),边框 `1px solid --border`,圆角 `--radius-md`,`font-mono text-sm`(实际是 12px),`line-height:1.6`,`white-space: pre-wrap`,内容可选中复制(`user-select: text`)。内容是该任务的逐行累计输出,**纯文本、不分行着色**(即便某一行是错误信息,也不单独标红——真实组件就是整段 `join("\n")` 渲染,没有逐行语义解析)。新日志追加后自动滚动到底部。日志文本本身是 Scoop CLI 原始输出,**不参与中英文翻译**,切换界面语言只影响面板自身的文案(标题、徽标、按钮),不影响日志内容。
- 空日志时显示一个占位符(如省略号),不显示空白。

## 7. 取消 / 关闭按钮设计指令(汇总)

| 按钮 | 出现位置 | 出现条件 | 颜色规则 | 点击效果 |
| --- | --- | --- | --- | --- |
| 关闭(X) | 面板头部右上角,全局唯一一个 | 始终存在,不因任务状态被禁用 | ghost(透明底,hover 浅灰底) | 整个面板隐藏;任何正在执行的任务**不受影响**继续在后台跑 |
| 取消 | 列表每一行内联,可同时出现多个 | 仅该行状态为"排队中"或"执行中" | **中性灰**(`--fg-muted` 系),**禁止 destructive 红色** | 该任务进入"已取消"终态;若取消的是"执行中"的任务,下一条排队中任务按创建顺序自动进入执行中(单 worker 顺序执行,不产生"失败"语义) |

## 8. 外部点击关闭行为(产品硬约束,必须还原)

面板打开时,点击面板范围之外的任意区域:

1. **关闭面板**(不是"提示是否关闭",直接关闭);
2. **吞掉这一次点击**,即底层被点到的元素(按钮、表格行、导航项等)**不会**响应这次点击——用户需要再点一次才能真正操作底层元素;
3. 例外:Radix Portal 渲染的对话框(`role="dialog"`)与 Sonner Toast 不受此规则影响,点击它们既不关闭面板也不被吞掉(实现上这两类元素在面板打开期间本就与面板互斥或叠加于更高层级,不构成"外部点击")。

这条行为在静态原型里通过真实的 `pointerdown`(捕获阶段关闭 + 标记吞掉下一次 `click`)+ `click`(捕获阶段消费该标记)双监听复刻,而不是简单一个 `click` 监听器就了事——顺序错了会导致"点一下背景按钮,面板关了,按钮还是被触发了"这种不符合硬约束的效果。

## 9. 多语言与无障碍

- 面板内全部文案(标题、状态徽标、取消/关闭按钮、空状态提示、日志区标题前缀"任务日志")均有 zh/en 两套,key 命名沿用 `jobs.*`/`common.*` 既有前缀,不新增命名风格。
- 关闭按钮需要 `aria-label`(对应 `common.close`);列表行可键盘聚焦(`tabIndex=0`)并支持 `Enter` 选中,与真实组件的可访问性处理一致。
- 中英文案长度差异会导致任务标题截断位置不同(如 "已失败(可重试)" 明显长于 "Failed (retryable)"),原型演示语言切换时应保证徽标区域不因文案变长而把取消按钮挤出面板宽度(即两套语言都要过一遍视觉检查)。

## 10. 视觉规范速查(引用设计系统,不重复定义)

字体、色板 oklch 数值、圆角、阴影、玻璃质感、微光效果的具体数值以 `docs/design/systems/web.md` 第 2/4/6/7/11 节为准;本页面在色彩使用上的唯一特殊规则是第 7 节的"取消按钮中性灰",除此之外沿用全局 token,不引入 P08 专属颜色。
