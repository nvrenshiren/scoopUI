## 1. 设计目标与场景定位

- P02 是 Boot Sequence(`flow` §2.6)"检测语言偏好"阶段、且判定为"未选择"时呈现的**唯一**画面(见 `flow` §2.5.3-1/2、§1.1 步骤 2-3)。用户此刻还没有任何界面语言偏好,是整个产品生命周期里**第一个**会被看到的画面(比 P01 的"检测中"更早出现)。
- 设计基调:极简二选一 + 品牌首因印象。不做多余装饰,不引入表单、列表或对话框;整屏只有一个决策点,决策后立即离场。
- 不复用 P03 主界面壳(无侧边栏、无顶栏)。Boot Sequence 的各子阶段(`init`/`language`/`detecting`/`not-found`/`installing`)在 `App.tsx` 中都是整屏铺满渲染;P02 与 P01 共享同一套"全屏居中、`bg-grid`+`bg-glow-top`背景"母版语言,但 P02 更轻——没有卡片容器包裹整体,只有图标、文案与两张选择卡直接居中排布。
- 因为此刻语言状态机(`flow` §2.5)处于"未选择",**标题下方的说明文案必须中英并列同屏呈现**,而不是按"当前语言"只显示一种语言——这是本页与产品内其它页面最大的差异点:其它页面文案都是"当前语言单语言展示",只有 P02 因为语言还未选定而双语并列。真实实现见 `src/pages/LanguagePick.tsx` 顶部注释:"首次选择前无语言偏好,双语并列呈现(此页即语言选择本身)"。

## 3. 关键区块设计指令

### 3.1 背景层(科技感,§2.1/§9)

- 整屏叠加 `.bg-grid`(32px 网格线,`--border` 约 8% 透明度)。
- 顶部约 40% 高度叠加 `.bg-glow-top`(以 `--primary-glow` 为中心的椭圆径向光晕,仅覆盖 `inset-x-0 top-0 bottom-[60%]` 区域),不要铺满全屏,否则会盖过下方选择卡片的可读性。
- 背景层 `pointer-events: none`,不得截获点击。

### 3.2 品牌区(图标徽标 + 主标题)

- 图标徽标:64×64,圆角 `radius-xl`(16px),背景 `--bg-elevated`,描边 **`1px solid var(--primary-glow)`**(注意:描边用的是半透明光晕色,不是实心 `--border` 或 `--primary`——这是本页专属的"发光徽标"处理,区别于侧边栏 Logo 徽标的做法),阴影 `0 0 24px var(--primary-glow)`,居中放一个终端 `>_` 意向图标,颜色 `--primary`。
- 主标题"Scoop GUI":`font-heading`,**32px**,字重 700,颜色 `--fg`。
  - ⚠️ **与设计系统 §4 字号尺度存在未对齐的差异**:设计系统把 `text-4xl`(2.25rem / 36px)定义为"首启页大标题"的标准字号,但 `LanguagePick.tsx` 真实实现用的是自定义 `text-[32px]`(32px),既不等于 36px 也不等于 30px 的 `text-3xl`。**本提示词与下游原型按真实实现的 32px 执行**,不要擅自改成 36px;这是设计系统条款与代码之间尚未对齐的已知差异,已在验收文档与本文档中一并记录,交由项目统一决定后续是调整设计系统措辞还是调整组件实现。

### 3.3 说明文案区(副标题 + 提示语)

- 副标题(14px,`--fg-muted`):中英并列同屏,用 `·` 分隔,不换行展示两套独立句子,不做语言切换。
- 提示语(12px,`--fg-subtle`):同样中英并列,传达"这不是一次性、不可逆的选择,之后仍可在设置中更改"。
- **这两行文案禁止各自拆成"当前语言单语言展示"**——参照第 1 节说明,本页是唯一允许双语同屏的页面,因为此刻语言状态机处于"未选择",没有"当前语言"可言。

### 3.4 语言选择卡片区

- 两张卡片左右并排,间距 16px(`space-4`),与说明文案区之间留白约 36px。
- 单卡规格:宽度固定 220px(高度随内容自适应),内边距 20px,圆角 `radius-lg`(12px),背景 `--bg-elevated`,描边 `1px solid --border`。
- 卡片内部纵向排列:语言名(`font-heading` 20px / 600)+ 一行该语言写就的简短预览说明(13px,`--fg-muted`)。这一行"预览说明"就是页面 PRD 里"语言预览"要求的落地方式——不需要额外的图片/截图预览,一句该语言自己的介绍文案即是预览。
- 右侧垂直居中放一个 `ChevronRight` 小箭头(18px,默认 `--fg-subtle`),暗示"点击即进入下一步"。
- **卡片本身就是可点击的整体目标**(原生 `<button>`,不是"卡片容器 + 内部单独按钮"两层结构),点击卡片内任意位置都触发选择。
- **点击 = 选择 = 确认,是同一个动作**,不设二级"确认"按钮或二次弹窗。详见第 5 节状态说明与验收文档的对应澄清。

## 4. 组件与 Design Token 映射表

| 设计元素 | 对应设计系统 token / 组件形态 | 备注 |
|---|---|---|
| 全屏背景 | `--bg`(§2)+ `.bg-grid` + `.bg-glow-top`(§2.1) | 与 P01"检测中"子态共用同一背景处理,视觉家族一致 |
| 图标徽标 | `--bg-elevated` + `radius-xl`(§6)+ `--primary-glow` 描边与光晕(§7) | 不套用 §8.2 标准 Card 规格(无实色 `--border`,改用光晕描边) |
| 主标题 | `font-heading` / 自定义 32px / 700 | 与 §4"text-4xl 首启页大标题"条款存在字号差异,见 3.2 |
| 副标题/提示语 | `font-body` 14px `--fg-muted` / 12px `--fg-subtle`(对应 §4 字号尺度 text-sm / text-xs) | 中英并列,不走单语言 `t()` 判定 |
| 选择卡片 | `--bg-elevated` + `1px solid --border` + `radius-lg`(§8.2 Card 变体) | 是"可点击的 Card",不是标准 `Button`(§8.1)variant |
| 卡片 hover/focus | `box-shadow: 0 0 0 1px var(--primary), 0 0 20px var(--primary-glow)` + `translateY(-2px)` | 呼应 §7 边框微光理念,但用 20px 模糊光晕而非 §7 给出的 4px 硬环,是同一"微光"理念下的合理变体 |
| 箭头图标 | `lucide-react ChevronRight` 18px,`--fg-subtle` → hover/focus 变 `--primary` | 纯装饰性意向提示,不单独可点击 |

## 5. 状态与交互反馈

| 状态 | 视觉表现 |
|---|---|
| 默认 | 如第 3 节所述,两张卡片视觉权重相同,不预设"推荐语言"倾向 |
| hover(鼠标悬停) | 卡片上浮约 2px + 描边变 `--primary` + 20px 主色光晕 + 箭头变 `--primary` |
| focus-visible(键盘 Tab 聚焦) | 与 hover 视觉一致(卡片是原生 `<button>`,Tab 可达,Enter/Space 直接触发选择),必须给出可见聚焦环,呼应设计系统 §9"键盘聚焦环必带" |
| 点击后 | **没有"已选中但未生效"的中间态**。点击即刻:①语言状态机从"未选择"进入对应的"已选中文/已选英文"并立即生效;②本机持久化写入开始;③Boot Sequence 立即推进到"检测 Scoop",P02 本身从画面上卸载,不停留、不显示 loading。因此 P02 自身不需要设计"提交中"视觉态 |
| 持久化失败(F19) | 不在 P02 页面内呈现;由跨页的 Sonner Toast(§8.10)在下一屏(检测中/P01/P03)上方弹出业务化提示,P02 无需为此设计 inline 错误态 |

## 6. 文案落位

本页文案**必须**逐字对应 `src/pages/LanguagePick.tsx` 的真实字面量,不要照抄 `src/i18n.ts` 字典里同名的 `lang.*` 词条——两者不一致(详见下方⚠️说明)。

| 区块 | 实际显示文案(逐字) |
|---|---|
| 主标题 | `Scoop GUI` |
| 副标题 | `选择界面语言 · Choose your language` |
| 提示语 | `之后可在设置中随时切换 · You can change it anytime in Settings` |
| 卡片 1 主文案 / 预览行 | `中文` / `简体中文界面` |
| 卡片 2 主文案 / 预览行 | `English` / `English interface` |

⚠️ **i18n 字典存在死码,设计与验收都不要以字典为准**:`src/i18n.ts` 里定义了 `lang.title`("选择界面语言"/"Choose your language")、`lang.subtitle`、`lang.zh`("中文"/"中文 (Chinese)")、`lang.en` 四个词条,但 `LanguagePick.tsx` 全文没有调用 `t()`/`tf()`,这四个词条在整个代码库里没有任何引用点。组件把双语并列文案直接硬编码在 JSX 里,文案内容与字典值也对不上(例如字典 `lang.zh` 的英文态是"中文 (Chinese)",但页面实际展示的第二行预览是"简体中文界面"/"English interface",是完全独立的一套文案)。这不是本次设计需要修的问题(不改字典、不改组件),只是提醒:**做本页原型和验收,文案来源 = 组件字面量,不是字典**。

## 7. 硬约束核对(摘自设计系统 §9,按本页适用性标注)

| 硬约束项 | 本页适用性 |
|---|---|
| 暗色默认 | 适用;本页始终尊重已持久化的 `theme`(理论上可能是亮色备选,见 `store.ts` 的 `boot()` 先应用主题再判断语言),但绝大多数首次启动场景下 `theme` 还是默认值 `dark`,原型以暗色为准 |
| 网格背景 / 微光(科技感) | 适用,见 3.1/3.4 |
| i18n `t('key')` | **本页是唯一例外**:见第 6 节,双语并列而非单语言 `t()` 判定,这是业务上合理的例外(语言尚未选定),不是违反硬约束 |
| 按钮禁用规则(长时操作中除取消外禁用) | 不适用:本页没有长时操作,也没有互斥的"进行中"态 |
| 对话框 / Toast 错误展示 | 不适用于本页内部(本页没有 inline 错误态);持久化失败走 Toast,但呈现在下一屏 |
| 键盘 Tab/Enter | 适用:两张卡片必须是可 Tab 聚焦、Enter/Space 直接触发的原生可交互元素 |
| 无第三方语言入口 | 适用且是本页核心边界:只能出现"中文"与"English"两张卡片,不得出现第三个选项、语言包导入入口或"自动检测系统语言"之类的额外入口(`project.md` §3.3、`flow` §2.5.4) |

## 8. 尺寸与响应式

- 窗口默认 1280×800,最小 1024×600(design-system §9 / `tauri.conf.json`)。
- 内容整体在窗口内水平垂直居中,不做栅格拉伸;窗口变大只增加四周留白,两张卡片宽度固定 220px,不随窗口宽度变化。
- 无独立滚动区域;整页内容高度远小于 600px 最小可用高度,不需要考虑滚动态。
