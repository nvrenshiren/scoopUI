设计与行为标注(非产品 UI 本身,供评审阅读)

  • 非 modal、无遮罩:面板本身不带任何背景遮罩,底层页面(此处以 P04 已装软件包列表模拟)保持完全可交互、不变暗。悬浮定位为窗口右下角(right 24px / bottom 56px),不是设计系统骨架图暗示的"居中浮层"——已按 JobsPanel.tsx 真实实现校正。
  • 列表 + 详情双区,不是单任务对话框:上方是全部任务的列表(本例 6 条,覆盖排队中/执行中/已成功/已失败(可重试)/已取消五态),下方详情区只展示"当前选中"的一条。
  • 自动跟随规则:在你手动点击任何一行之前,详情区会自动跟随"当前执行中"的任务——试试不要点列表,观察 git 安装完成后详情区如何自动切到刚开始执行的 ffmpeg。手动点击过某一行后,详情区"钉住"该行,不再自动跳转。
  • 进度条恒为不确定态(扫动动画),没有百分比数值——Scoop CLI 不产出结构化进度,产品也没有编造一个,仅在该任务为"排队中/执行中"时渲染。
  • 取消 ≠ 失败:取消按钮用中性灰(--fg-muted),不用 destructive 红色;只出现在"排队中/执行中"的任务行内联位置,可同时存在多个。点击 nodejs 或 ffmpeg 行的"取消"试试。
  • 关闭按钮不受忙碌状态影响:面板头部右上角 X 任何时候都可点击关闭,任务不会因此中断——这是"非 modal"的核心体现。关闭后右下角出现"任务"重新打开入口(镜像真实侧边栏的任务导航项)。
  • 点击外部关闭并吞掉该次点击:面板打开时点击背景任意位置(如侧边栏、表格),面板会关闭,但不会触发被点到的控件——需要再点一次才会真正生效。试着点一下背景里的"检查更新"按钮观察效果。
  • 面板本身不提供"重试"按钮:失败任务(python)只显示徽标,没有内联重试操作;"重试"通过返回来源页重新发起同类动作实现——下方 mock 表格里 python 行的"更新"按钮是可点的,点击会在面板里生成一条全新排队中任务,演示这条真实路径。
  • F16(协助安装 Scoop)不在本组件范围内:P01 首次启动协助安装的进度/日志由 BootView.tsx 自行绘制成居中卡片,不经过这个右下角面板,本原型不重复呈现该场景。
  • 日志内容不参与翻译:切换右上角语言只影响面板文案(标题/徽标/按钮),日志区文本始终是 Scoop CLI 原始输出。
本页为静态设计原型,交互均为前端模拟(vanilla JS),不连接真实 Tauri IPC;计时器与状态跃迁仅用于演示真实组件的状态机与选中逻辑。
