## 1. 验收范围与前提

本文档验收 P05(浏览与搜索软件包页)覆盖的功能点:F01(可装部分)、F02、F04(出口)、F05、F17(出口)、F19。不覆盖:P07 详情对话框自身内容、P08 进度对话框自身内容、P04 已装列表页——均见各自页面验收文档。

前提条件(除非用例另有说明):

- 本机 Scoop 可用(`Scoop 自身 = 可用`),已进入主界面(`Boot Sequence = 进入主界面`)。
- 用户已通过 P03 侧边栏导航到"浏览软件包"。

## 2. 可装列表展示(F01 可装部分)

| ID | 前置条件 | 操作 | 预期结果 |
|---|---|---|---|
| P05-TC01 | 首次进入本页(`availableLoaded` 为假) | 无 | 页面立即发起一次无查询参数的可装软件包列表读取;读取完成前,搜索框保持可见可输入,其下方区域显示加载态(旋转图标 +"正在读取全部可装软件包(首次较慢,取决于桶大小)…") |
| P05-TC02 | 可装列表已加载完成,查询为空 | 无 | 展示已添加桶范围内的全部可装软件包,不做任何过滤 |
| P05-TC03 | 同一会话内已加载过一次,再次导航离开又返回本页(未点刷新) | 无 | 不重新发起读取,直接展示已缓存的列表(`availableLoaded` 为真时跳过重复请求) |
| P05-TC04 | 任意时刻 | 点击 PageHeader 右侧"刷新"按钮 | 按钮立即禁用并将图标切换为旋转态;强制重新读取可装列表(即使之前已加载过);读取完成后按钮恢复可点击、图标恢复常态 |
| P05-TC05 | 可装软件包数量超过前端渲染上限(500 条) | 无 | 表格只渲染前 500 条;Footer 文案变为"显示 500 / {total} 条 · 输入关键字缩小范围" |

## 3. 关键字搜索(F02)

| ID | 前置条件 | 操作 | 预期结果 |
|---|---|---|---|
| P05-TC10 | 列表已加载 | 在搜索框输入一个与部分包名匹配的子串 | 表格即时(无延迟、无防抖)只展示"名称"字段包含该子串的行(大小写不敏感);过滤过程不产生新的可装列表读取请求(基于已加载数据在前端过滤) |
| P05-TC11 | 同上 | 输入的关键字只能匹配到某些行的版本号或来源桶,不匹配任何名称 | 该关键字不命中任何行——搜索只按"名称"字段过滤,不匹配版本号 / 来源桶 |
| P05-TC12 | 已输入关键字且已过滤出部分结果 | 清空搜索框全部内容 | 列表恢复展示已添加桶范围内的全部可装软件包 |
| P05-TC13 | 已输入关键字 | 关键字与任何软件包名称都不匹配 | 表格区域替换为空态提示("没有匹配的软件包" /"换个关键字,或到"桶管理"添加更多桶"),搜索框本身保持可见且可继续编辑 |
| P05-TC14 | 数据仍在加载中(loading 为真) | 在搜索框输入内容 | 输入本身不被阻止(搜索框未被禁用);列表区域此时仍展示加载态而非过滤结果,直至数据到达、loading 转为假后,才按当前搜索框内容过滤展示 |

## 5. 执行安装(F05 → P08 出口)

| ID | 前置条件 | 操作 | 预期结果 |
|---|---|---|---|
| P05-TC30 | 某行为"可装未装"(未安装,且当前该包名无进行中任务) | 点击该行"安装"按钮 | 按钮点击后**没有二次确认弹窗**,立即调用入队接口;入队成功后任务面板(P08)自动展开 |
| P05-TC31 | 上一步入队成功,任务处于"排队中"或"执行中" | 观察该行 | 状态列显示 `info` 徽标(旋转图标 + 当前任务类型文案,如"安装");操作列的安装按钮仍然渲染,但置为禁用态(按钮不消失,只置灰) |
| P05-TC32 | 该任务最终进入"已成功" | 无(等待事件回调) | 状态列切换为 `success` 徽标"已安装";操作列不再渲染安装按钮;出现 4 秒自动消失的成功提示 Toast;仅对该软件包做单包状态刷新,不触发全量已装列表 / 过期检查刷新 |
| P05-TC33 | 该任务最终进入"已失败(可重试)" | 无(等待事件回调) | 出现停留 6 秒的失败提示 Toast;任务面板自动展开供查看日志;该行**恢复为"可装未装"的默认呈现**(状态列清空、安装按钮重新可点击)——本页表格不单独持久显示"安装失败"标记,失败信息只通过 Toast 与任务面板呈现;用户直接再次点击同一枚安装按钮即可重试,构成本页的"重试入口" |
| P05-TC34 | 该任务被用户在任务面板中主动取消 | 无(等待事件回调) | 出现"已取消"提示 Toast;该行恢复为"可装未装"默认呈现,视觉表现与失败路径相同(表格本身不区分"失败"与"取消") |
| P05-TC35 | 安装成功 | 切换到 P04 已装软件包列表页查看 | 应能看到该软件包出现在已装列表中(验证 F05 业务结果:可装未装 → 已装(最新版)) |

## 9. 与真实实现的核对说明(登记差异,不代表缺陷)

以下几点是撰写验收用例过程中,对照页面 PRD 与 `src/pages/BrowseView.tsx` 实际实现后发现的出入。按"以代码为准"原则,验收标准已依据真实实现编写;记录如下供知悉,不需要为此改动 PRD 或设计系统等上游契约文件:

1. 页面 PRD 场景 5 描述为"用户对某个可装软件包选择安装**并确认**",而实际实现点击"安装"按钮即立即入队,**没有**二次确认弹窗(与卸载操作的确认弹窗形成对比,后者在 `InstalledView.tsx` 中有独立的 `ConfirmDialog`)。第 5 节验收用例按无确认弹窗的实际行为编写。
2. 页面 PRD 验收要点"安装失败时,用户应看到失败原因与重试入口"——实际实现中,"重试入口"就是同一枚"安装"按钮在任务失败后自然恢复为可点击状态,而非独立的"重试"按钮;表格行本身不持久显示"失败"标记(见 P05-TC33)。
3. 表格行的"打开详情"目前只支持鼠标点击,不支持纯键盘操作(见 P05-TC22),已登记为已知差异,不计入本次验收通过 / 失败判定。
4. 搜索(F02)在本页的实现是纯前端对已全量加载数据的过滤,不会因为输入关键字而产生新的 `scoop_search` 调用,即使该 IPC 命令本身支持按关键字查询。
5. 表格不展示 `binaries`(可执行文件)字段,尽管它是 `SearchResult` 数据结构的一部分——该信息只在 P07 详情对话框中呈现。
