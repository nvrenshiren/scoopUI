## 1. 范围与前置说明

- **覆盖范围**:P03 主界面壳本身——Sidebar(导航 + 底部功能区)、主区路由(`view` → 具体子视图组件)、跨视图常驻的横切能力(F18 单实例聚焦、F19 友好提示的承载)。
- **不覆盖**:P04/P05/P06/P09/P07/P08 各自内部的业务验收(过期比对、搜索过滤、桶增删、详情对话框、进度呈现等),这些留待各自页面的验收文档。本文档只在"这些页面如何被 P03 路由到"的边界上与它们发生关系。
- **测试前置条件**:应用启动序列(`flows/scoop-gui.md` §2.6)已进入"进入主界面",对应 `src/store.ts` 的 `boot === "ready"`。
- **真相源标注方式**:凡本文档结论与 `docs/prd/pages/web/scoop-gui/P03-主界面壳.md` 第 5 节"验收要点"或 `docs/design/systems/web.md` 表述不一致之处,均在对应用例后以"**〔差异登记〕**"标出,并给出真实代码依据;不代为修改上游文档。

## 4. 导航跳转(F01 框架能力 + 出口 P04/P05/P06/P09)

| 用例 ID | 操作 | 预期结果 |
|---|---|---|
| AC-P03-13 | 点击"已装软件" | 主区渲染 `InstalledView`(P04);该导航项进入选中态 |
| AC-P03-14 | 点击"浏览软件包" | 主区渲染 `BrowseView`(P05);该导航项进入选中态;`BrowseView` 首次挂载时会自行触发 `refreshAvailable()`(见 `BrowseView.tsx` 挂载副作用),即"首次浏览"可能出现短暂加载态,这是子页面职责,P03 仅负责挂载该组件 |
| AC-P03-15 | 点击"桶管理" | 主区渲染 `BucketsView`(P06);该导航项进入选中态 |
| AC-P03-16 | 点击"设置" | 主区渲染 `SettingsView`(P09);该导航项进入选中态 |
| AC-P03-17 | 连续切换 4 个导航项 | 每次切换后只有当前项处于选中态,不出现"多项同时选中"或"无项选中" |

**〔差异登记〕AC-P03-18** ——各子视图内部输入状态在导航切换后的保留情况:

- 前提:在"已装软件"页的过滤输入框中键入文本(`InstalledView.tsx` 的本地 `filter` state),或在"浏览软件包"页搜索框键入关键字(`BrowseView.tsx` 的本地 `query` state),或在"桶管理"页打开"添加自定义桶"对话框并输入了桶名(`BucketsView.tsx` 的本地 `addOpen`/`addName`/`addRepo` state)。
- 操作:通过侧边导航切换到其它主区,再切回原主区。
- **实际结果**(以 `src/App.tsx` 的 `VIEWS[view]` 条件渲染 + 上述三个页面组件源码为准):P03 主区按 `view` 值整树替换渲染(同一插槽内组件类型变化,触发 React 卸载旧组件、挂载新组件),因此离开视图时该视图的本地 `useState` 全部被销毁;切回后以上过滤文本/搜索关键字/对话框草稿均**重置为初始值**,不会保留用户此前输入。
- 页面 PRD `docs/prd/pages/web/scoop-gui/P03-主界面壳.md` 第 5 节验收要点第 4 条要求"用户在 P04/P05/P06 之间切换时,各页面已输入的搜索或筛选条件不应无故丢失",与当前实现**不一致**。本用例按真实实现如实登记为"当前不满足",不作为通过/不通过的强制判据,留待 developer/PM 决定是否需要把相关本地状态提升到 `store.ts`(或引入 keep-alive 式的多树并存渲染),或修订该条验收要点。

## 6. 常驻任务入口(真实实现;上游文档未直接声明的入口)

**〔差异登记〕**:`src/App.tsx` 的 `Shell` 组件在 Sidebar 底部(弹性留白之后)渲染了一个持久化的"任务"(Jobs)导航按钮,点击后切换 `jobsPanelOpen`,展开/收起 `JobsPanel`(P08)浮层;按钮右侧带进行中任务数徽标(`activeJobCount`)。

- 模块 PRD `docs/prd/modules/scoop-gui.md` 第 3 节页面清单里,P08 的"入口"被定义为"从 P04/P05/P06/P07 触发 F05/F06/F07/F08/F11/F12/F16",并未提及 P03 本身也是 P08 的常驻触发入口。
- 页面 PRD `docs/prd/pages/web/scoop-gui/P03-主界面壳.md` 第 2 节功能清单同样未把"任务面板入口"列为 P03 的功能项。
- 但真实实现里,该入口是 P03 Shell 级别的常驻元素,与当前 `view` 无关(不随导航切换而消失),本文档按真实实现登记以下验收用例:

| 用例 ID | 操作 | 预期结果 |
|---|---|---|
| AC-P03-21 | 有任务处于"排队中"或"执行中" | 任务入口右侧显示带旋转动画的信息色徽标,数字为进行中任务数(`selectBusyTargets`/`activeJobCount` 计数口径:`state === "queued" \|\| "running"`) |
| AC-P03-22 | 点击任务入口 | `JobsPanel` 浮层展开;再次点击或点击浮层外部区域(且不落在 radix portal/toast 上)使其收起 |
| AC-P03-23 | 在任一导航项之间切换 | 任务入口的展开/收起状态与徽标计数不受 `view` 切换影响(因为 `JobsPanel` 渲染在 `App()` 顶层,与 `Shell` 是兄弟节点,不在 `view` 路由的主区之内) |

> 本节差异不构成"实现错误"的判定——只是提醒:若后续要把 `docs/design/systems/web.md` §8.7/§10.1 中"Sidebar 底部 = Theme toggle + 中英切换"的骨架与真实实现的"Sidebar 底部 = 任务入口"对齐,需要统一裁决(二选一或并存),不在本文档职责范围内。

依据 `src-tauri/src/lib.rs`:`tauri_plugin_single_instance::init` 回调对已存在的 `"main"` 窗口依次执行 `show()` → `unminimize()` → `set_focus()`。

| 用例 ID | 前置 | 操作 | 预期结果 |
|---|---|---|---|
| AC-P03-28 | 已有一个 scoop-gui 实例在运行(任意 `view`,任意窗口状态) | 再次启动 scoop-gui(如双击安装后的快捷方式,或命令行再次执行) | **不**创建第二个可独立操作的窗口/进程;已有窗口被显示、取消最小化并获得焦点 |
| AC-P03-29 | 已有实例被最小化 | 再次启动 | 已有窗口从最小化恢复并置于前台 |
| AC-P03-30 | 已有实例此前导航到"桶管理"(或已展开任务面板) | 再次启动触发聚焦 | 由于是**同一进程/同一 React 树被重新置于前台**(而非重新挂载),`view`、`jobsPanelOpen`、各子页面本地状态均保持触发前的原样——这与第 4 节"AC-P03-18 导航切换会重置本地状态"是两种不同场景,不应混淆:AC-P03-18 描述的是应用内切换导航(组件卸载/重挂载),本用例描述的是窗口失焦后再聚焦(组件从未卸载) |
| AC-P03-31 | — | 测试方法说明 | 本节验收必须基于打包后的 Tauri 可执行文件(或至少两个连到同一 Tauri 后端的运行时实例)进行;`npm run dev` 的纯浏览器 mock 模式没有操作系统进程/单实例概念,无法验证本节用例 |

## 13. 文档范围声明

本文档只回答"P03 主界面壳本身"的可测试验收用例:入口门禁、Sidebar 结构与状态、导航路由、跨视图常驻能力(任务面板、语言切换即时性)、单实例聚焦、错误提示呈现、负向验收。

**本文档不涉及**:P04/P05/P06/P07/P08/P09 各自内部业务规则的验收(过期比对细节、搜索匹配规则、桶增删流程、详情字段展示、任务日志滚动等)——这些由各自页面的验收文档覆盖;任何 UI 视觉规范本身的定义(见设计系统文档);任何 IPC/Rust 侧实现细节的验收(除 F18 单实例这一在 P03 直接可观察的行为外)。

本文档第 4、6、11、12 节登记的"差异"均为真实源码核对后的客观记录,不修改、不覆盖 `docs/prd/pages/web/scoop-gui/P03-主界面壳.md`、`docs/prd/modules/scoop-gui.md`、`docs/design/systems/web.md` 的既有条款;取舍由这些文档的统一维护者决定。
